Chapter 13: Variant Calling

~90 min Advanced GATK, VCF, SNPs, Indels, Annotation, bcftools

What is Variant Calling?

Variant calling identifies positions where a sample's DNA differs from the reference genome. These differences include:

  • SNPs (Single Nucleotide Polymorphisms) — A→G at one position
  • Indels — Small insertions or deletions (1-50 bp)
  • Structural variants — Large rearrangements (>50 bp): deletions, duplications, inversions, translocations
  • Copy number variants (CNVs) — Gains or losses of large genomic segments

The GATK Best Practices Pipeline

Sorted BAM (from BWA alignment)
  ↓ Mark Duplicates (GATK/Picard)
Deduped BAM
  ↓ Base Quality Score Recalibration (BQSR)
Recalibrated BAM
  ↓ Variant Calling (HaplotypeCaller)
Raw gVCF (per sample)
  ↓ Joint Genotyping (GenomicsDBImport + GenotypeGVCFs)
Raw VCF (all samples)
  ↓ Variant Filtering (VQSR or hard filters)
Filtered VCF
  ↓ Annotation (SnpEff, VEP)
Annotated VCF

Step 1: Mark Duplicates

gatk MarkDuplicates \
    -I sorted.bam \
    -O dedup.bam \
    -M dedup_metrics.txt \
    --REMOVE_DUPLICATES false   # Mark, don't remove

samtools index dedup.bam

PCR duplicates are copies of the same original DNA molecule. They inflate allele counts and can make heterozygous sites look homozygous. Marking them tells downstream tools to ignore them.

Step 2: Base Quality Score Recalibration (BQSR)

# Build recalibration model using known variant sites
gatk BaseRecalibrator \
    -I dedup.bam \
    -R reference.fa \
    --known-sites dbsnp.vcf.gz \
    --known-sites known_indels.vcf.gz \
    -O recal_table

# Apply recalibration
gatk ApplyBQSR \
    -I dedup.bam \
    -R reference.fa \
    --bqsr-recal-file recal_table \
    -O recal.bam

Step 3: Call Variants

# Per-sample calling (produces gVCF)
gatk HaplotypeCaller \
    -I recal.bam \
    -R reference.fa \
    -ERC GVCF \
    -O sample.g.vcf.gz

# Joint genotyping across all samples
gatk GenomicsDBImport \
    -V sample1.g.vcf.gz -V sample2.g.vcf.gz -V sample3.g.vcf.gz \
    --genomicsdb-workspace-path genomicsdb \
    -L intervals.list

gatk GenotypeGVCFs \
    -R reference.fa \
    -V gendb://genomicsdb \
    -O raw_variants.vcf.gz

Step 4: Filter Variants

# Separate SNPs and indels
gatk SelectVariants -V raw_variants.vcf.gz --select-type-to-include SNP -O raw_snps.vcf.gz
gatk SelectVariants -V raw_variants.vcf.gz --select-type-to-include INDEL -O raw_indels.vcf.gz

# Hard filtering (when VQSR not possible — e.g., small sample sizes)
gatk VariantFiltration -V raw_snps.vcf.gz \
    --filter-expression "QD < 2.0" --filter-name "LowQD" \
    --filter-expression "MQ < 40.0" --filter-name "LowMQ" \
    --filter-expression "FS > 60.0" --filter-name "HighFS" \
    --filter-expression "SOR > 3.0" --filter-name "HighSOR" \
    --filter-expression "MQRankSum < -12.5" --filter-name "LowMQRS" \
    --filter-expression "ReadPosRankSum < -8.0" --filter-name "LowRPRS" \
    -O filtered_snps.vcf.gz

VCF Format — In Depth

##fileformat=VCFv4.2
##INFO=<ID=DP,Number=1,Type=Integer,Description="Total Depth">
##INFO=<ID=AF,Number=A,Type=Float,Description="Allele Frequency">
##FORMAT=<ID=GT,Number=1,Type=String,Description="Genotype">
##FORMAT=<ID=DP,Number=1,Type=Integer,Description="Read Depth">
##FORMAT=<ID=AD,Number=R,Type=Integer,Description="Allelic Depths">
#CHROM  POS     ID      REF  ALT  QUAL  FILTER  INFO           FORMAT       SAMPLE1     SAMPLE2
chr1    12345   rs123   A    G    99    PASS    DP=60;AF=0.5   GT:DP:AD    0/1:30:15,15  0/0:28:28,0
chr1    67890   .       CT   C    50    PASS    DP=40;AF=0.25  GT:DP:AD    0/1:20:15,5   0/0:18:18,0

Genotype Field (GT)

0/0  = Homozygous reference (both alleles = REF)
0/1  = Heterozygous (one REF, one ALT)
1/1  = Homozygous alternate (both alleles = ALT)
1/2  = Heterozygous with two different ALT alleles
./.  = Missing genotype (not enough data)

Working with VCF: bcftools

# Statistics summary
bcftools stats filtered.vcf.gz > stats.txt

# Extract specific fields
bcftools query -f '%CHROM\t%POS\t%REF\t%ALT\t%QUAL\t[%GT]\n' filtered.vcf.gz

# Filter: only PASS variants
bcftools view -f PASS filtered.vcf.gz -o pass_only.vcf.gz

# Filter by quality and depth
bcftools view -i 'QUAL>30 && INFO/DP>10' filtered.vcf.gz

# Extract a region
bcftools view filtered.vcf.gz chr1:1000000-2000000

# Count variants
bcftools view -H filtered.vcf.gz | wc -l

# Merge VCFs from multiple samples
bcftools merge sample1.vcf.gz sample2.vcf.gz -o merged.vcf.gz

Variant Annotation

# SnpEff — predict functional effects
snpEff ann GRCh38.105 filtered_snps.vcf.gz > annotated.vcf

# Categories of variant effects:
# HIGH:     frameshift, stop gained, splice donor/acceptor
# MODERATE: missense variant
# LOW:      synonymous variant
# MODIFIER: intron, intergenic, UTR

# Ensembl VEP (Variant Effect Predictor) — more comprehensive
vep --input_file filtered.vcf.gz \
    --output_file annotated.txt \
    --cache --species homo_sapiens \
    --assembly GRCh38 \
    --everything
Variant Calling Tips
  • Always use joint calling (gVCF → GenomicsDBImport → GenotypeGVCFs) for cohort studies
  • VQSR requires ≥30 exomes or ≥1 whole genome. Use hard filters for smaller datasets
  • Validate important variants with IGV visualization
  • For somatic (cancer) variants, use Mutect2 instead of HaplotypeCaller
