What is a Neural Network?

A neural network is a function composed of layers. Each layer applies a linear transformation (a weighted sum) followed by a non-linear activation function. Stacking multiple layers allows the network to learn increasingly abstract representations of the input data.

Input layer:   20,000 genes (or 50 PCA components)
      ↓ (linear transform + activation)
Hidden layer 1: 512 neurons
      ↓ (linear transform + activation)
Hidden layer 2: 256 neurons
      ↓ (linear transform + activation)
Output layer:  33 neurons (one per cancer type, softmax activation)

The activation function introduces non-linearity. Without it, stacking linear layers would still be a linear model. The most common activation in modern networks is ReLU (Rectified Linear Unit): f(x) = max(0, x). It is simple, computationally efficient, and works remarkably well in practice.

The output layer uses softmax, which converts raw scores into probabilities that sum to 1 across all classes. The predicted class is the one with the highest probability.

Building a Neural Network Classifier in PyTorch

PyTorch is the standard deep learning framework in research. It gives you explicit control over every component of the network and training loop, which makes it easier to understand what is happening and to extend to custom architectures later in the course.

import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset
import numpy as np

# Convert numpy arrays to PyTorch tensors
X_train_tensor = torch.FloatTensor(X_train_pca)
y_train_tensor = torch.LongTensor(y_train)
X_test_tensor  = torch.FloatTensor(X_test_pca)
y_test_tensor  = torch.LongTensor(y_test)

# Create DataLoaders for batched training
train_dataset = TensorDataset(X_train_tensor, y_train_tensor)
train_loader  = DataLoader(train_dataset, batch_size=256, shuffle=True)

# Define the network architecture
class CancerClassifier(nn.Module):
    def __init__(self, input_dim, hidden_dim, output_dim, dropout=0.3):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_dim // 2, output_dim)
        )

    def forward(self, x):
        return self.network(x)

# Instantiate model
n_pcs      = X_train_pca.shape[1]   # 50
n_classes  = len(np.unique(y_train)) # 33
model      = CancerClassifier(input_dim=n_pcs, hidden_dim=256,
                               output_dim=n_classes)

# Loss function and optimiser
criterion = nn.CrossEntropyLoss()
optimiser = optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)

Training Loop

def train_epoch(model, loader, criterion, optimiser):
    model.train()
    total_loss, correct, total = 0, 0, 0
    for X_batch, y_batch in loader:
        optimiser.zero_grad()
        outputs = model(X_batch)
        loss    = criterion(outputs, y_batch)
        loss.backward()
        optimiser.step()
        total_loss += loss.item()
        correct    += (outputs.argmax(1) == y_batch).sum().item()
        total      += y_batch.size(0)
    return total_loss / len(loader), correct / total

def evaluate(model, X, y):
    model.eval()
    with torch.no_grad():
        outputs = model(X)
        preds   = outputs.argmax(1)
        acc     = (preds == y).float().mean().item()
    return acc, preds.numpy()

# Train for 50 epochs
train_losses, train_accs, test_accs = [], [], []
for epoch in range(50):
    loss, train_acc = train_epoch(model, train_loader, criterion, optimiser)
    test_acc, _     = evaluate(model, X_test_tensor, y_test_tensor)
    train_losses.append(loss)
    train_accs.append(train_acc)
    test_accs.append(test_acc)
    if (epoch + 1) % 10 == 0:
        print(f"Epoch {epoch+1}: loss={loss:.4f}, "
              f"train={train_acc:.3f}, test={test_acc:.3f}")

Learning Curves

Plotting training and test accuracy over epochs is one of the most useful diagnostics in deep learning. If training accuracy keeps rising while test accuracy plateaus or declines, the model is overfitting. If both plateau early at a low value, the model is underfitting.

On TCGA with 50 PCA inputs, a two-layer neural network typically achieves around 94-95% accuracy. This is marginally better than logistic regression but requires substantially more code, more hyperparameter tuning, and more training time. The improvement is real but modest. We are not yet in the regime where neural network complexity is clearly justified over linear methods.

Autoencoders — Learning to Compress

An autoencoder is a neural network trained to reconstruct its own input. It has two parts: an encoder that compresses the input to a low-dimensional representation, and a decoder that reconstructs the original input from that compressed representation.

Input (20,000 genes)
       ↓  Encoder
Latent space (50-100 dimensions)
       ↓  Decoder
Reconstruction (20,000 genes)

Training objective: minimise reconstruction error
               (output should look like input)

The bottleneck forces the network to learn a compact representation that captures the most important structure in the data. Everything that cannot be reconstructed from the bottleneck must be discarded — so the network learns to keep what matters and discard what does not.

This is fundamentally different from PCA. PCA finds linear combinations of features. An autoencoder can find non-linear combinations. If the important structure in gene expression data is non-linear — and in many cases it is — autoencoders can capture it where PCA cannot.

Building an Autoencoder in PyTorch

class Autoencoder(nn.Module):
    def __init__(self, input_dim, latent_dim):
        super().__init__()

        # Encoder: compress input to latent space
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 512),
            nn.ReLU(),
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim)
        )

        # Decoder: reconstruct input from latent space
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 512),
            nn.ReLU(),
            nn.Linear(512, input_dim)
        )

    def forward(self, x):
        z = self.encoder(x)        # compress
        x_recon = self.decoder(z)  # reconstruct
        return x_recon, z          # return both reconstruction and latent code

    def encode(self, x):
        return self.encoder(x)

# Use raw scaled expression (not PCA) as input to the autoencoder
# The autoencoder will learn its own compression
X_train_raw = torch.FloatTensor(X_train_scaled)
X_test_raw  = torch.FloatTensor(X_test_scaled)

ae = Autoencoder(input_dim=X_train_scaled.shape[1], latent_dim=64)
ae_optimiser = optim.Adam(ae.parameters(), lr=1e-3)
recon_criterion = nn.MSELoss()  # minimise reconstruction error

# Training loop for autoencoder
train_dataset_ae = TensorDataset(X_train_raw)
train_loader_ae  = DataLoader(train_dataset_ae, batch_size=256, shuffle=True)

for epoch in range(100):
    ae.train()
    total_loss = 0
    for (X_batch,) in train_loader_ae:
        ae_optimiser.zero_grad()
        x_recon, _ = ae(X_batch)
        loss = recon_criterion(x_recon, X_batch)
        loss.backward()
        ae_optimiser.step()
        total_loss += loss.item()
    if (epoch + 1) % 20 == 0:
        print(f"Epoch {epoch+1}: recon loss = {total_loss/len(train_loader_ae):.4f}")

Using Autoencoder Representations for Classification

Once trained, the encoder produces a 64-dimensional latent representation for each sample. You can use these representations as features for downstream classification — just like you used PCA components in Chapters 7 and 8.

# Extract latent representations
ae.eval()
with torch.no_grad():
    Z_train = ae.encode(X_train_raw).numpy()
    Z_test  = ae.encode(X_test_raw).numpy()

print(f"Latent representation shape: {Z_train.shape}")
# (n_train_samples, 64)

# Train logistic regression on autoencoder representations
lr_ae = LogisticRegression(C=1.0, max_iter=1000,
                            multi_class='multinomial', solver='lbfgs')
lr_ae.fit(Z_train, y_train)
y_pred_ae = lr_ae.predict(Z_test)

Visualising the Latent Space

The latent space is 64-dimensional, but you can visualise it in 2D using PCA or UMAP applied to the latent vectors. If the autoencoder learned meaningful representations, cancer types should cluster in latent space.

The Full Comparison So Far

You now have four methods to compare on the same TCGA task:

Method Approx. accuracy Training time Interpretable?
PCA + Logistic Regression ~93% Seconds Yes
PCA + Random Forest ~94% 2 minutes Partially
PCA + Neural Network ~94-95% 5 minutes No
Autoencoder + Logistic Regression ~93-95% 10 minutes No

The pattern emerging is clear. Adding complexity brings diminishing returns on this task. The performance difference between the simplest method and the most complex is a few percent. Whether that matters depends on the biological and clinical context. For some applications it does. For most research applications, the simple method wins.

The autoencoder's value is not primarily in classification accuracy. It is in what the latent space reveals — structure in the data that PCA might miss because it is non-linear. In Chapter 10, variational autoencoders take this further by learning a structured, continuous latent space that enables interpolation between samples and generative capabilities.

Reality Check
  • Does the neural network significantly outperform logistic regression on your specific task?
  • Did you check learning curves for overfitting? Training accuracy much higher than test accuracy is a red flag.
  • Does the autoencoder latent space show cancer type separation? If not, the representations are not biologically meaningful.
  • Is the complexity justified by the performance gain?
