The Question This Entire Course Has Been Building Toward

Since Chapter 1 you have been asked the same question at the end of every chapter: does the complexity actually help? You have seen logistic regression nearly match neural networks on TCGA. You have seen PCA compete with scVI on clean single-cell data. You have seen benchmarks showing that simple baselines outperform foundation models on many standard tasks.

Now you test it yourself. Not on toy data. Not on a pre-processed demonstration. On a real benchmark dataset used in published evaluation papers, with a fair experimental design, measuring what actually matters.

The task is cell type annotation. The representations are PCA, scVI, and Geneformer. The classifier is the same for all three. The only thing that changes is the embedding. At the end, you will have an answer — your answer, computed on your machine — to the question that defines this course.

The Dataset

We use the PBMC benchmark dataset from Luecken et al. (2022), the single-cell integration benchmarking (scIB) study published in Nature Methods. This dataset was collected specifically to benchmark representation learning and integration methods, making it ideal for evaluating foundation models.

Dataset:    scIB PBMC Benchmarking Dataset
Source:     Luecken et al., Nature Methods 2022
Cells:      ~14,000 PBMCs
Donors:     10 donors (batch variable)
Cell types: CD4 T, CD8 T, NK, B cell, Monocyte, Dendritic, Plasma
Platform:   10x Chromium
Access:     figshare.com/ndownloader/files/24539842
Format:     AnnData (.h5ad)
Size:       ~150 MB

This dataset has several properties that make it ideal for the capstone. It has real batch effects across 10 donors, so scVI's batch correction can actually demonstrate its value. The cell types are well-characterised with high-confidence labels. And because it was explicitly used to benchmark representation methods in a published study, you can compare your results to Table 1 of the Luecken et al. paper.

Experimental Design

The design is deliberately simple. Three representations, one classifier, one evaluation protocol. The simplicity is the point — it isolates the representation as the only variable.

For each representation method (PCA, scVI, Geneformer):

  1. Extract embeddings for all cells
  2. Split into 80% train / 20% test (stratified by cell type)
  3. Train a k-Nearest Neighbour classifier (k=15) on train embeddings
  4. Predict cell types on test embeddings
  5. Compute: Accuracy, Macro-F1, per-class F1
  6. Measure: wall-clock runtime, memory usage
  7. Visualise: UMAP of embeddings coloured by cell type and donor

Record all results in a comparison table.
Final question: which method would you deploy in a real analysis, and why?

import scanpy as sc
import scvi
import numpy as np
import pandas as pd
import torch
import time

from sklearn.neighbors import KNeighborsClassifier

import matplotlib.pyplot as plt

Step 1: Load and Inspect the Data

import urllib.request

# Download the scIB PBMC benchmark dataset
url  = "https://figshare.com/ndownloader/files/24539842"
path = "pbmc_scib.h5ad"
print("Downloading dataset (~150 MB)...")
urllib.request.urlretrieve(url, path)

# Load
adata = sc.read_h5ad(path)
print(adata)
print("\nCell type distribution:")
print(adata.obs['cell_type'].value_counts())
print("\nDonor distribution:")
print(adata.obs['batch'].value_counts())
print(f"\nGenes: {adata.n_vars}")
print(f"Cells: {adata.n_obs}")
Before any analysis — inspect your data

Check that cell type labels are present, that batch labels are present, that the counts look like integers (raw counts, not normalised). This dataset should be ready to use after loading, but always verify before training any model.

Step 2: Preprocessing

# Store raw counts for scVI (needs integer counts)
adata.layers['counts'] = adata.X.copy()

# Standard preprocessing for PCA
sc.pp.highly_variable_genes(adata, n_top_genes=2000,
                              subset=False, flavor='seurat_v3',
                              batch_key='batch')
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
sc.pp.scale(adata, max_value=10)

# Encode cell type labels
from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
y  = le.fit_transform(adata.obs['cell_type'])

print(f"Classes: {le.classes_}")
print(f"Encoded: {np.unique(y)}")

Step 3: Representation 1 — PCA

print("=" * 50)
print("METHOD 1: PCA")
print("=" * 50)

start_time = time.time()
tracemalloc.start()

# PCA on highly variable genes
sc.tl.pca(adata, n_comps=50, use_highly_variable=True)
X_pca = adata.obsm['X_pca']

pca_time   = time.time() - start_time
pca_memory = tracemalloc.get_traced_memory()[1] / 1e6  # MB
tracemalloc.stop()

print(f"PCA embedding shape: {X_pca.shape}")
print(f"Time:   {pca_time:.1f} seconds")
print(f"Memory: {pca_memory:.1f} MB")

# Train/test split
X_train_pca, X_test_pca, y_train, y_test = train_test_split(
    X_pca, y, test_size=0.2, random_state=42, stratify=y
)

# Classify
knn_pca = KNeighborsClassifier(n_neighbors=15, metric='cosine', n_jobs=-1)
knn_pca.fit(X_train_pca, y_train)
y_pred_pca = knn_pca.predict(X_test_pca)

acc_pca = accuracy_score(y_test, y_pred_pca)
f1_pca  = f1_score(y_test, y_pred_pca, average='macro')
print(f"PCA + kNN  |  Accuracy: {acc_pca:.3f}  |  Macro-F1: {f1_pca:.3f}")
print(classification_report(y_test, y_pred_pca, target_names=le.classes_))

Step 4: Representation 2 — scVI

print("=" * 50)
print("METHOD 2: scVI")
print("=" * 50)

start_time = time.time()
tracemalloc.start()

# Setup scVI with batch correction
adata_scvi = adata.copy()
adata_scvi.X = adata_scvi.layers['counts']  # use raw counts

scvi.model.SCVI.setup_anndata(
    adata_scvi,
    layer='counts',
    batch_key='batch'
)

model_scvi = scvi.model.SCVI(adata_scvi, n_latent=20, n_layers=2)
model_scvi.train(max_epochs=200, early_stopping=True,
                  early_stopping_patience=20, batch_size=256)

X_scvi = model_scvi.get_latent_representation()
adata.obsm['X_scVI'] = X_scvi

scvi_time   = time.time() - start_time
scvi_memory = tracemalloc.get_traced_memory()[1] / 1e6
tracemalloc.stop()

print(f"scVI embedding shape: {X_scvi.shape}")
print(f"Time:   {scvi_time:.1f} seconds")
print(f"Memory: {scvi_memory:.1f} MB")

# Classify with same split
X_train_scvi = X_scvi[np.where(np.isin(np.arange(len(y)), train_idx))[0]]
X_test_scvi  = X_scvi[np.where(np.isin(np.arange(len(y)), test_idx))[0]]

knn_scvi = KNeighborsClassifier(n_neighbors=15, metric='cosine', n_jobs=-1)
knn_scvi.fit(X_train_scvi, y_train)
y_pred_scvi = knn_scvi.predict(X_test_scvi)

acc_scvi = accuracy_score(y_test, y_pred_scvi)
f1_scvi  = f1_score(y_test, y_pred_scvi, average='macro')
print(f"scVI + kNN  |  Accuracy: {acc_scvi:.3f}  |  Macro-F1: {f1_scvi:.3f}")

from geneformer import TranscriptomeTokenizer, EmbExtractor
from datasets import load_from_disk

GENEFORMER_MODEL = "ctheodoris/Geneformer"  # update to latest stable release

start_time = time.time()

# Step 1: Convert AnnData to Geneformer input format
# Geneformer requires a specific loom or dataset format
# Save as loom first
import loompy
adata_gf = adata.copy()
adata_gf.X = adata_gf.layers['counts'].astype(int)

# Tokenise for Geneformer
tk = TranscriptomeTokenizer(
    {"cell_type": "cell_type", "batch": "batch"},
    nproc=4
)
tk.tokenize_data(
    data_directory="./geneformer_input/",
    output_directory="./geneformer_tokens/",
    output_prefix="pbmc_scib",
    file_format="loom"
)

# Step 2: Extract embeddings
embex = EmbExtractor(
    model_type="Pretrained",
    num_classes=0,
    emb_mode="cell",
    cell_emb_style="mean_pool",
    filter_data=None,
    max_ncells=None,
    emb_layer=-1,
    emb_label=["cell_type"],
    forward_batch_size=100,
    nproc=4
)

embs = embex.extract_embs(
    model_directory=GENEFORMER_MODEL,
    input_data_file="./geneformer_tokens/pbmc_scib.dataset",
    output_directory="./geneformer_embeddings/",
    output_prefix="pbmc_scib"
)

X_geneformer = embs.values
adata.obsm['X_geneformer'] = X_geneformer

gf_time   = time.time() - start_time
gf_memory = tracemalloc.get_traced_memory()[1] / 1e6
tracemalloc.stop()

print(f"Geneformer embedding shape: {X_geneformer.shape}")
print(f"Time:   {gf_time:.1f} seconds")

What the Literature Expects You to Find

Based on the scIB benchmarking paper and subsequent foundation model evaluations, here is what you are likely to see — though your exact numbers will vary:

  • PCA achieves reasonable but not state-of-the-art F1, especially on rare cell types
  • scVI improves over PCA on overall performance and shows clearer separation between donors in UMAP — the batch correction is visible
  • Geneformer approximately matches scVI on overall Macro-F1, sometimes better, sometimes worse by a few percent
  • PCA takes seconds; scVI takes minutes; Geneformer takes significantly longer and requires more memory
  • The UMAP coloured by donor shows the key difference: PCA separates by donor, scVI mixes donors by cell type, Geneformer varies

If you find that PCA achieves 85% Macro-F1, scVI achieves 88%, and Geneformer achieves 89% — is that 4-point improvement over PCA worth 100x the compute time? There is no universal answer. It depends on whether you are doing exploratory analysis on a laptop or a production clinical tool. This is the judgement the course has been preparing you to make.

Compare your results to the literature
