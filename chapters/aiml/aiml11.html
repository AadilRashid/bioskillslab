The Shift to Single-Cell

Run this chapter in Google Colab (GPU recommended)

Everything you have built so far — PCA, logistic regression, random forests, autoencoders, VAEs — was applied to bulk RNA-seq data from TCGA, where each sample is the averaged expression of millions of cells. Now the course shifts to single-cell RNA-seq, where each measurement comes from one individual cell.

This shift changes everything. Bulk RNA-seq gives you one data point per patient. Single-cell RNA-seq gives you thousands or millions of data points per patient, each representing the transcriptional state of one cell. The challenge is no longer just classification or dimensionality reduction — it is handling data with properties that standard machine learning methods were not designed for.

scVI (single-cell Variational Inference) is the tool that solves these challenges. It is the most widely used deep learning framework for single-cell analysis, with thousands of citations and an active development ecosystem. Understanding it deeply will prepare you for the foundation models in Chapter 12.

Why Standard Methods Fail on Single-Cell Data

Single-cell RNA-seq data has three properties that make it different from bulk data and that break standard assumptions.

Sparsity. Most genes are not detected in most cells. A typical single-cell dataset might be 90-95% zeros. A cell that actually expresses a gene may show zero counts simply because not enough RNA molecules were captured in the sequencing library. This is called dropout. Standard normalisation and log transformation handle this poorly — the zeros are not biologically meaningful absences, they are technical noise.

Count data with overdispersion. Single-cell counts follow a negative binomial distribution, not a Gaussian. The variance is much larger than the mean (overdispersion), and this varies between genes and between cells. Models that assume Gaussian noise will misfit the data.

Batch effects at scale. Single-cell experiments are enormously sensitive to technical variation. Samples processed on different days, in different labs, with different chemistries, or at different sequencing depths show systematic differences that can be larger than the biological signal. In a multi-sample study, batch effects are almost always present and must be corrected before any biological interpretation.

The scVI Model

scVI (Lopez et al., 2018) addresses all three challenges simultaneously. It is a VAE specifically designed for single-cell count data. Instead of using mean squared error as the reconstruction loss — which assumes Gaussian noise — scVI uses a negative binomial likelihood that matches the actual statistical distribution of single-cell counts.

The model architecture:

Input: raw count matrix (cells x genes)
       ↓
Encoder: fully connected layers
       ↓
Latent space z: continuous low-dimensional representation
       ↓
Decoder: conditioned on z AND batch label
       ↓
Reconstruction: negative binomial parameters (mean, dispersion)

Training objective:
  Maximise evidence lower bound (ELBO)
  = Reconstruction likelihood - KL divergence

The crucial innovation is conditioning the decoder on the batch label. This allows the model to learn batch-specific technical variation separately from the biological signal in the latent space. The latent representation z captures biology; the decoder with batch conditioning reconstructs the batch-specific observation. This separation is what enables batch correction: to compare cells across batches, you use the latent space z directly, ignoring the batch-specific reconstruction.

Loading and Preparing PBMC Data

We use the same PBMC 3k dataset from Project 4, but now through the lens of deep learning.

import scanpy as sc
import scvi
import numpy as np
import matplotlib.pyplot as plt

# Load PBMC 3k
adata = sc.datasets.pbmc3k()
print(adata)
# AnnData object with n_obs x n_vars = 2700 x 32738

# Standard preprocessing (same as Project 4)
sc.pp.filter_cells(adata, min_genes=200)
sc.pp.filter_genes(adata, min_cells=3)
adata.var['mt'] = adata.var_names.str.startswith('MT-')
sc.pp.calculate_qc_metrics(adata, qc_vars=['mt'], inplace=True)
adata = adata[adata.obs.pct_counts_mt < 5].copy()

# Select highly variable genes
sc.pp.highly_variable_genes(adata, n_top_genes=2000,
                              subset=True, flavor='seurat_v3',
                              batch_key=None)

print(f"Cells: {adata.n_obs}, Genes: {adata.n_vars}")

# scVI uses raw counts directly — do NOT normalise or log-transform
# The model handles count statistics internally
adata.layers['counts'] = adata.X.copy()  # store raw counts
Raw counts only

scVI requires raw integer counts as input. Do not normalise or log-transform the data before passing it to scVI. The model's negative binomial likelihood is designed to work with raw counts and will produce wrong results if given normalised data. This is the opposite of what you do for PCA or logistic regression.

Setting Up and Training scVI

# Register the AnnData object with scVI
# Tell it which layer contains raw counts and which column is batch
scvi.model.SCVI.setup_anndata(
    adata,
    layer='counts',
    # batch_key='batch'  # uncomment if you have batch information
)

# Create the model
model = scvi.model.SCVI(
    adata,
    n_latent=20,          # latent space dimensions
    n_layers=2,           # encoder/decoder depth
    n_hidden=128,         # hidden layer size
    dropout_rate=0.1
)

print(model)

# Train the model
model.train(
    max_epochs=200,
    batch_size=128,
    early_stopping=True,
    early_stopping_patience=20
)

# Plot training history
model.history['elbo_train'].plot()
plt.title('scVI Training ELBO')
plt.xlabel('Epoch')
plt.show()

Extracting Latent Representations

After training, extract the latent representation for every cell. This 20-dimensional vector per cell is the batch-corrected biological representation that you use for all downstream analysis.

# Get the latent embedding for all cells
adata.obsm['X_scVI'] = model.get_latent_representation()

print(f"Latent embedding shape: {adata.obsm['X_scVI'].shape}")
# (2638, 20)  — one 20-dim vector per cell

# UMAP on the scVI latent space (not on raw expression)
sc.pp.neighbors(adata, use_rep='X_scVI', n_neighbors=15)
sc.tl.umap(adata)

# Leiden clustering on the scVI graph
sc.tl.leiden(adata, resolution=0.5, key_added='leiden_scvi')

# Visualise
sc.pl.umap(adata, color=['leiden_scvi'], title='scVI Latent Space')

Comparing scVI to Standard PCA Pipeline

The key question from Chapter 6 applies here: does scVI actually do better than the standard PCA pipeline from Project 4? To answer this fairly, you need to compare them on a task with a known ground truth.

# Run standard pipeline for comparison
adata_pca = adata.copy()
sc.pp.normalize_total(adata_pca, target_sum=1e4)
sc.pp.log1p(adata_pca)
sc.pp.scale(adata_pca, max_value=10)
sc.tl.pca(adata_pca, svd_solver='arpack')
sc.pp.neighbors(adata_pca, n_pcs=40)
sc.tl.umap(adata_pca)
sc.tl.leiden(adata_pca, resolution=0.5, key_added='leiden_pca')

# Compare UMAPs side by side
fig, axes = plt.subplots(1, 2, figsize=(14, 6))
sc.pl.umap(adata,     color='leiden_scvi', ax=axes[0],
           show=False, title='scVI clusters')
sc.pl.umap(adata_pca, color='leiden_pca',  ax=axes[1],
           show=False, title='PCA clusters')
plt.tight_layout()
plt.show()

# Compare known marker gene expression
markers = ['CD3D', 'CD14', 'MS4A1', 'GNLY', 'FCER1A', 'PPBP']
sc.pl.umap(adata,     color=markers, ncols=3, title='scVI')
sc.pl.umap(adata_pca, color=markers, ncols=3, title='PCA')

For the PBMC 3k dataset without batch effects, scVI and PCA typically give very similar results. The cell type clusters are nearly identical in both UMAPs. The marker gene patterns are equally clear. This is an important validation of Chapter 6's message: when the data is clean and the task is straightforward, simpler methods work just as well as complex ones.

The scenario where scVI genuinely outperforms PCA is when batch effects are present. With multiple samples from different donors or different sequencing runs, PCA will show samples clustering by batch while scVI will show them mixing by cell type. This is the real value of scVI.

Batch Correction with scVI

# Simulating multi-batch data for demonstration
# In practice, batch_key points to a column in adata.obs

# Suppose we have data from two donors
adata.obs['donor'] = ['donor1' if i < 1300 else 'donor2'
                       for i in range(adata.n_obs)]

# Retrain with batch correction
scvi.model.SCVI.setup_anndata(adata, layer='counts',
                               batch_key='donor')
model_batch = scvi.model.SCVI(adata, n_latent=20)
model_batch.train(max_epochs=200, early_stopping=True)

# Extract batch-corrected latent space
adata.obsm['X_scVI_batch'] = model_batch.get_latent_representation()

# The latent space should now mix donors by cell type, not separate them
sc.pp.neighbors(adata, use_rep='X_scVI_batch')
sc.tl.umap(adata)
sc.pl.umap(adata, color=['donor', 'leiden_scvi'],
           title=['Donor (should mix)', 'Cell type (should separate)'])

Beyond PBMC: What scVI Enables

scVI is not just a batch correction tool. The scvi-tools ecosystem has grown into a comprehensive framework for single-cell analysis with several extensions:

  • scANVI — extends scVI with semi-supervised cell type annotation using a small set of labelled cells
  • totalVI — handles CITE-seq data with simultaneous RNA and protein measurements
  • MultiVI — integrates RNA, ATAC-seq, and protein from multi-omic experiments
  • destVI — deconvolution of spatial transcriptomics data

All of these share the same core VAE architecture from scVI, extended with modality-specific likelihood functions and conditioning mechanisms. Understanding scVI gives you the foundation to use the entire ecosystem.

Reality Check
  • Does scVI produce clearly better clusters than PCA on your data? If not, PCA is sufficient.
  • Do you actually have batch effects? If all samples were processed identically, scVI's batch correction adds nothing.
  • Did you check that you passed raw counts, not normalised data, to scVI?
  • Did you validate clusters against known marker genes regardless of which method you used?
