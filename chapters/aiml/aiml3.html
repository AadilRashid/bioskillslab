Chapter 3: Classification, Regression and Clustering

~35 min Beginner Classification, Regression, Clustering, Supervised, Unsupervised

The Biological Question

Before choosing any algorithm, you need to answer one question: what kind of output do you want?

Three scenarios from real biology:

  • A patient's tumour is either lung cancer, breast cancer, or colon cancer — which one? (a category)
  • What is the predicted survival time of this patient in months? (a number)
  • These 10,000 cells — do they naturally group into distinct populations? (unknown groups)

These three scenarios correspond to the three fundamental tasks in machine learning: classification, regression, and clustering. Understanding which task you are solving determines every subsequent decision — which algorithm to use, which metrics to evaluate, and how to interpret results.

Classification — Predicting a Category

Classification is the task of assigning an input to one of a fixed set of categories.

Input                          Output (category)
─────                          ─────────────────
Gene expression profile   →    Cancer subtype (lung / breast / colon)
Cell gene expression      →    Cell type (T cell / B cell / monocyte)
DNA sequence              →    Regulatory element (yes / no)
Patient genomic profile   →    Treatment responder (yes / no)
Variant                   →    Pathogenic / Benign / Uncertain

When the output has exactly two categories (tumour vs normal, pathogenic vs benign) it is called binary classification. When there are three or more categories (cancer subtype) it is called multi-class classification.

How classification works intuitively

Imagine plotting each tumour sample in a space where each axis is a gene's expression. Samples of the same cancer type tend to cluster together. Classification draws a decision boundary — a line or curve that separates the groups. New samples that land on one side are classified as type A; the other side, type B.

In practice you have 20,000 genes — 20,000 dimensions — not 2. The decision boundary exists in a space you cannot visualise. But the concept is identical.

Classification algorithms you will use in this course
  • Logistic Regression — simple, interpretable, often surprisingly competitive (Chapter 8)
  • Random Forest — robust, handles many features well, provides feature importance (Chapter 8)
  • Neural Networks — powerful for complex patterns, needs more data (Chapter 9)
  • Geneformer / scGPT — foundation models pretrained on millions of cells (Chapter 12)

Regression — Predicting a Number

Regression is the task of predicting a continuous numerical value rather than a category.

Input                          Output (number)
─────                          ───────────────
Gene expression profile   →    Survival time (months)
DNA sequence features     →    Gene expression level
Genomic variants          →    Protein abundance
Cell state features       →    Pseudotime (developmental stage)
Multi-omics profile       →    Drug response score (IC50)

The difference from classification is the nature of the output. Survival time is not a category — it is a continuous value that can be 3.2 months, 47.8 months, or any number in between. Predicting it requires regression.

When does it matter which you use?

Using the wrong task type is a common mistake. If you bin a continuous outcome into categories — "low survival" vs "high survival" — you discard information and reduce statistical power. If you treat a category as a number — assigning 1 to lung, 2 to breast, 3 to colon — you introduce a false ordering that confuses the model.

Match the algorithm to the output type.

Clustering — Discovering Groups Without Labels

Classification and regression both require labels — you need to know the correct answer for your training examples. Clustering does not. It finds natural groupings in data without being told what the groups are.

Input                          Output (discovered groups)
─────                          ──────────────────────────
10,000 single cells       →    8 cell type clusters
500 tumour samples        →    4 molecular subtypes
1,000 patients            →    3 patient stratification groups
Protein interaction data  →    Functional modules

Clustering is unsupervised — there are no labels. The algorithm finds structure based on similarity in the feature space. This is what happens when you run UMAP on single-cell data and see distinct blobs — those blobs are clusters discovered by the algorithm.

Clusters are not guaranteed to be biology

Clustering finds groups that are mathematically distinct in your feature space. Those groups may correspond to real biological cell types — or they may reflect technical variation, sequencing depth differences, or batch effects. Always validate clusters against known markers before claiming biological discovery.

The Three Tasks Side by Side

Task Output type Labels needed? Biological example
Classification Category Yes Cancer subtype, cell type, pathogenic variant
Regression Continuous number Yes Survival time, gene expression level, drug IC50
Clustering Group assignments No Cell populations, tumour subtypes, patient groups

Where Does PCA Fit?

You encountered PCA in the Statistics for Biology course. PCA is not classification, regression, or clustering — it is dimensionality reduction. It compresses high-dimensional data into fewer dimensions while preserving as much variation as possible.

PCA is typically used as a preprocessing step before clustering or classification:

Raw data (20,000 genes)
        ↓
PCA (reduce to 50 principal components)
        ↓
Clustering (find cell type groups)
        ↓
Classification (train a model on the clusters)

In single-cell analysis this exact pipeline — PCA then clustering then UMAP visualisation — is run millions of times every week in labs worldwide. Chapter 7 applies it to TCGA data and Chapter 11 applies it to single-cell data.

Choosing the Right Task

Before building any model, ask:

  1. What is my output? A category → classification. A number → regression. Unknown groups → clustering.
  2. Do I have labels? Yes → supervised (classification or regression). No → unsupervised (clustering).
  3. How many output categories? Two → binary classification. More → multi-class.
  4. Is a simpler method sufficient? A t-test, PCA, or differential expression analysis might answer your question without any model at all.

Reality Check
  • Does your biological question actually require machine learning, or would a simpler statistical test suffice?
  • If you found clusters — have you validated them against known biology or just assumed they are real?
  • If you are doing classification — where did your labels come from, and could they be wrong?
