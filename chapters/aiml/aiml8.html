The First Real Models

In Chapter 7 you prepared the TCGA data and confirmed through PCA that cancer types separate in expression space. Now you train your first classifiers. This chapter builds the two models that should always come before anything more complex: logistic regression and random forest.

These are not "simple" models in a dismissive sense. They are the gold standard baselines that any more complex model must beat to justify its additional complexity. In many genomics tasks, they win outright. You need to know what they are, how to train them properly, and how to interpret what they tell you about the biology.

Logistic Regression — The Interpretable Baseline

Despite its name, logistic regression is a classification algorithm. It finds a linear decision boundary in feature space by learning a weight for each feature — in our case, a weight for each principal component from Chapter 7.

The model calculates a weighted sum of the features, passes it through a sigmoid function to produce a probability between 0 and 1, and classifies based on that probability. For multi-class problems like TCGA with 33 cancer types, it extends to one-vs-rest or softmax classification.

The key advantage of logistic regression is interpretability. The learned weights tell you which features (principal components, or genes if you skip PCA) drive the predictions. A positive weight means higher values push toward the positive class. A negative weight means higher values push toward the negative class. You can look at the model and understand what it learned.

from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report, accuracy_score
from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay
import matplotlib.pyplot as plt

# Train logistic regression on PCA-reduced TCGA data
# C is the inverse of regularisation strength — smaller C = more regularisation
lr = LogisticRegression(
    C=1.0,
    max_iter=1000,
    multi_class='multinomial',
    solver='lbfgs',
    random_state=42
)
lr.fit(X_train_pca, y_train)

# Evaluate on test set
y_pred_lr = lr.predict(X_test_pca)
print(f"Accuracy: {accuracy_score(y_test, y_pred_lr):.3f}")
print("\nPer-class performance:")
print(classification_report(y_test, y_pred_lr,
                             target_names=le.classes_))

On TCGA Pan-Cancer data with 50 PCA components, logistic regression typically achieves around 90-95% accuracy across 33 cancer types. That number alone should make you pause before reaching for a neural network. A linear model, on compressed features, in under a minute of training time, correctly classifies 9 out of 10 cancer samples from gene expression alone.

Confusion Matrix

The accuracy number summarises everything into one figure. The confusion matrix shows you exactly where the model succeeds and fails — which cancer types are being confused with each other. This is where the biology lives.

# Confusion matrix
cm = confusion_matrix(y_test, y_pred_lr)
fig, ax = plt.subplots(figsize=(14, 12))
disp = ConfusionMatrixDisplay(confusion_matrix=cm,
                               display_labels=le.classes_)
disp.plot(ax=ax, xticks_rotation=90, colorbar=False)
plt.title('Logistic Regression — TCGA Cancer Type Confusion Matrix')
plt.tight_layout()
plt.show()

When you examine the confusion matrix, you will find that most errors are biologically sensible. Kidney renal clear cell carcinoma (KIRC) and kidney renal papillary carcinoma (KIRP) are confused with each other — they are both kidney cancers with overlapping expression profiles. Stomach adenocarcinoma (STAD) and esophageal carcinoma (ESCA) overlap — they are anatomically adjacent and share molecular features. The model's mistakes are not random; they reflect genuine biological similarity between cancer types.

Random Forests — Robust Non-Linear Classification

Random forests build a large number of decision trees, each trained on a random subset of the training samples and a random subset of the features. Each tree votes on the class, and the forest predicts the majority vote. The randomness is the key — it prevents any single tree from overfitting and makes the ensemble far more robust than any individual tree.

Random forests handle several challenges that are common in genomics. They work well when the number of features is large relative to the number of samples. They are robust to outlier samples. They do not require feature scaling. And crucially, they provide feature importance scores that tell you which genes (or PCA components) were most useful for classification.

from sklearn.ensemble import RandomForestClassifier
import numpy as np

# Train random forest
rf = RandomForestClassifier(
    n_estimators=500,     # number of trees
    max_features='sqrt',  # features per split (sqrt of total)
    min_samples_leaf=2,   # minimum samples per leaf node
    n_jobs=-1,            # use all CPU cores
    random_state=42
)
rf.fit(X_train_pca, y_train)

# Evaluate
y_pred_rf = rf.predict(X_test_pca)
print(f"Accuracy: {accuracy_score(y_test, y_pred_rf):.3f}")
print("\nPer-class performance:")
print(classification_report(y_test, y_pred_rf,
                             target_names=le.classes_))

# Feature importance
importances = rf.feature_importances_
plt.figure(figsize=(10, 4))
plt.bar(range(50), importances)
plt.xlabel('Principal Component')
plt.ylabel('Feature Importance')
plt.title('Random Forest Feature Importances — Top 50 PCs')
plt.show()

Random forests on TCGA typically achieve similar or slightly higher accuracy than logistic regression — around 92-96%. The performance difference between the two is often smaller than the variation between runs, which itself is a meaningful finding: for this particular task, model complexity beyond logistic regression adds marginal value.

Feature Importance — What Drives the Classification?

The feature importances from random forest tell you which principal components carry the most information for distinguishing cancer types. But PCA components are linear combinations of all genes, which makes them hard to interpret biologically. To get back to genes, you can examine the PCA loadings:

# Which genes contribute most to the top PCs?
# pca.components_ has shape (n_components, n_genes)
# Each row is a PC, each column is a gene's contribution

top_pc = 0  # look at PC1
gene_loadings = pd.Series(
    pca.components_[top_pc],
    index=gene_names  # your list of gene names
).abs().sort_values(ascending=False)

print(f"Top 20 genes driving PC{top_pc+1}:")
print(gene_loadings.head(20))

# Combine with RF importance:
# Which genes drive the PCs that the forest finds most important?
top_pcs = np.argsort(importances)[::-1][:5]  # top 5 important PCs
key_genes = set()
for pc in top_pcs:
    top_genes = pd.Series(
        np.abs(pca.components_[pc]),
        index=gene_names
    ).nlargest(10).index
    key_genes.update(top_genes)

print(f"\nKey genes driving classification:")
for g in sorted(key_genes): print(f"  {g}")

When you run this on TCGA, the genes that emerge are biologically coherent. Tissue-specific markers dominate: EPCAM for epithelial cancers, PTPRC (CD45) for haematopoietic tumours, KRT genes for squamous cell carcinomas. The model has independently rediscovered the molecular signatures that pathologists and cancer biologists have known about for decades. This is good validation — the model is learning real biology, not technical artifacts.

Cross-Validation — Getting Robust Estimates

A single train/test split gives you one performance estimate. Cross-validation gives you a distribution of estimates across multiple splits, which tells you not just how well the model performs but how stable that performance is.

from sklearn.model_selection import cross_val_score, StratifiedKFold

# 5-fold stratified cross-validation
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

# Logistic regression CV
lr_scores = cross_val_score(lr, X_train_pca, y_train,
                             cv=cv, scoring='accuracy', n_jobs=-1)
print(f"LR CV accuracy: {lr_scores.mean():.3f} +/- {lr_scores.std():.3f}")

# Random forest CV
rf_scores = cross_val_score(rf, X_train_pca, y_train,
                             cv=cv, scoring='accuracy', n_jobs=-1)
print(f"RF CV accuracy: {rf_scores.mean():.3f} +/- {rf_scores.std():.3f}")

# Per-class F1 via cross-validation
from sklearn.model_selection import cross_val_predict
y_pred_cv = cross_val_predict(rf, X_train_pca, y_train, cv=cv)
print("\nCross-validated classification report:")
print(classification_report(y_train, y_pred_cv, target_names=le.classes_))

Comparing the Two Models

After running both models, you should have something like this:

Model                    Accuracy   Training time   Interpretable?
─────                    ────────   ─────────────   ──────────────
Logistic Regression       ~93%      ~10 seconds     Yes (weights)
Random Forest             ~94%      ~2 minutes      Partially (importances)
Difference                ~1%       12x slower      Less interpretable

The random forest is marginally better. But is a 1% accuracy improvement worth 12x longer training time and less interpretability? For a research baseline, probably not. For a clinical tool where every percentage point matters, possibly yes.

This comparison also previews the question that runs through the entire course. In Chapter 9 we add a neural network. In Chapter 10 we add an autoencoder. In Chapter 12 we use Geneformer. At each step we ask the same question: does the additional complexity justify the additional cost?

The baseline you just built is not trivial

93% accuracy on 33-class cancer type prediction from gene expression alone, using a model that trains in seconds, is genuinely impressive. Many papers that report superior deep learning performance compare against much weaker baselines. Keep these numbers in mind when you read claims that foundation models are necessary for cancer classification tasks.

Reality Check
  • Did logistic regression and random forest achieve similar performance? If so, the simpler model wins.
  • Do the confusion matrix errors make biological sense? Confused cancer types should be biologically similar.
  • Do the important genes match known cancer biology? If the model found random genes, something may be wrong.
  • Would you trust either model on samples from a different institution or a different sequencing platform?
