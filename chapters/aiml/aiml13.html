From Cells to Sequences

Chapters 11 and 12 focused on gene expression — what is being transcribed in individual cells. This chapter shifts to the DNA sequence itself — what determines whether genes are transcribed at all. Regulatory genomics is the field that asks how the genome is controlled, and it is one of the areas where AI has made some of its most genuine and reproducible advances in biology.

The key insight driving this chapter is simple: DNA is a sequence of characters (A, T, C, G), and language models are very good at understanding sequences. The same transformer architecture that models the relationships between words in a sentence can model the relationships between nucleotide positions in a DNA sequence — with one critical difference. In language, words are separated by spaces and their boundaries are clear. In DNA, the relevant units (transcription factor binding motifs, splice sites, regulatory elements) are embedded in a continuous string of 3 billion characters with no delimiters. The challenge is tokenisation: how do you break DNA into meaningful units?

Tokenising DNA — The k-mer Approach

The simplest and most widely used approach is k-mer tokenisation. Rather than treating each nucleotide as a token, you use overlapping windows of k nucleotides. For k=6, the sequence ATCGATCG becomes: ATCGAT, TCGATC, CGATCG — overlapping hexamers that capture local sequence context.

Sequence:  A T C G A T C G A T
k=3 tokens: ATC, TCG, CGA, GAT, ATC, TCG, GAT
k=6 tokens: ATCGAT, TCGATC, CGATCG, GATCGA, ATCGAT

Vocabulary size:
  k=3: 4^3 = 64 possible trimers
  k=4: 4^4 = 256 possible tetramers
  k=6: 4^6 = 4096 possible hexamers

The choice of k involves a trade-off. Small k captures local sequence features but misses longer-range context. Large k captures more context but has an exponentially growing vocabulary and sparser representations. k=6 is the standard choice for most DNA language models, striking a balance between context and tractability.

DNABERT — BERT for the Genome

DNABERT (Ji et al., Bioinformatics 2021) applies the BERT pretraining paradigm to human DNA sequences. It tokenises the genome into overlapping k-mers and uses masked language modelling — randomly masking k-mer tokens and training the model to predict them from context — to learn general representations of DNA sequence.

The pretrained model can then be fine-tuned for specific tasks: predicting transcription factor binding sites, identifying splice sites, annotating promoters and enhancers, or predicting the regulatory impact of variants.

In 2023, DNABERT-2 significantly improved on the original by using byte-pair encoding (BPE) tokenisation — the same approach used in modern language models — instead of fixed k-mers. BPE learns a data-driven vocabulary of variable-length DNA tokens, which better captures the non-uniform information content of different genomic regions.

    "zhihan1996/DNABERT-2-117M",
    trust_remote_code=True

model.eval()

# Example: embed a DNA sequence
sequence = "ATCGATCGATCGATCGATCGATCGATCGATCG"
inputs   = tokenizer(sequence, return_tensors='pt')

with torch.no_grad():
    outputs  = model(**inputs)
    # outputs.last_hidden_state: (batch, seq_len, hidden_size)
    # Pool across sequence positions to get one vector per sequence
    embedding = outputs.last_hidden_state.mean(dim=1)  # (1, 768)
    print(f"Sequence embedding shape: {embedding.shape}")

Fine-tuning DNABERT-2 for Promoter Prediction

from transformers import AutoModelForSequenceClassification
from torch.utils.data import Dataset, DataLoader
import torch.nn as nn

# Binary classification: is this sequence a promoter?
class SequenceDataset(Dataset):
    def __init__(self, sequences, labels, tokenizer, max_length=512):
        self.encodings = tokenizer(sequences, truncation=True,
                                    padding=True, max_length=max_length,
                                    return_tensors='pt')
        self.labels = torch.LongTensor(labels)

    def __len__(self):
        return len(self.labels)

    def __getitem__(self, idx):
        item = {k: v[idx] for k, v in self.encodings.items()}
        item['labels'] = self.labels[idx]
        return item

Enformer — Long-Range Regulatory Prediction

DNABERT models sequences up to a few thousand base pairs. But gene regulation operates at scales of hundreds of thousands of base pairs. Enhancers can act on genes 500,000 base pairs away. Understanding gene expression from sequence requires a model that can look across these distances.

Enformer (Avsec et al., Nature Methods 2021) was designed specifically for this challenge. It takes 196,608 base pairs of DNA sequence as input and predicts 5,313 genomic tracks — gene expression across tissues, chromatin accessibility, histone modifications, transcription factor binding, and more — from the sequence alone.

The architecture combines a convolutional stem (for local feature extraction) with a transformer tower (for long-range interactions). The attention mechanism allows the model to directly attend to distant enhancers when predicting gene expression at a given position. This long-range attention is the key capability that earlier CNN-based models like DeepSEA lacked.

pip install enformer-pytorch

import enformer_pytorch
from enformer_pytorch import Enformer
import torch

# Load pretrained Enformer
enformer = Enformer.from_pretrained('EleutherAI/enformer-official-rough')
enformer.eval()

# Predict from sequence (196,608 bp = 196kb window)
# Input: one-hot encoded DNA, shape (batch, 4, 196608)
# A=0, C=1, G=2, T=3

def one_hot_encode(sequence):
    mapping = {'A': 0, 'C': 1, 'G': 2, 'T': 3, 'N': 0}
    encoded = torch.zeros(4, len(sequence))
    for i, base in enumerate(sequence.upper()):
        if base in mapping:
            encoded[mapping[base], i] = 1
    return encoded

# Load a 196kb sequence around a gene of interest (e.g. from Ensembl)
seq = "ATCG..."  # 196,608 characters
x   = one_hot_encode(seq).unsqueeze(0)  # (1, 4, 196608)

with torch.no_grad():
    predictions = enformer(x)
    # predictions['human']: shape (1, 896, 5313)
    # 896 bins of 128bp each covering the central 114,688 bp
    # 5313 tracks: CAGE, DNase, ChIP-seq, etc.

# Extract gene expression prediction (CAGE tracks)
cage_tracks = predictions['human'][:, :, :675]  # first 675 tracks are CAGE
print(f"CAGE predictions shape: {cage_tracks.shape}")

Variant Effect Prediction with Enformer

One of Enformer's most powerful applications is predicting the regulatory effect of variants. For a given SNP, you run Enformer on the reference sequence and the alternative sequence and compare the predicted tracks. Variants that substantially change predicted gene expression or chromatin accessibility are likely to be functionally important.

def predict_variant_effect(ref_seq, alt_seq, enformer):
    x_ref = one_hot_encode(ref_seq).unsqueeze(0)
    x_alt = one_hot_encode(alt_seq).unsqueeze(0)

    with torch.no_grad():
        pred_ref = enformer(x_ref)['human']
        pred_alt = enformer(x_alt)['human']

    # Log fold change for each track at each position
    log_fc = torch.log2(pred_alt + 1e-6) - torch.log2(pred_ref + 1e-6)
    return log_fc

# The variant with the largest absolute log_fc across CAGE tracks
# is predicted to have the largest regulatory effect
effect = predict_variant_effect(ref_seq, alt_seq, enformer)
max_effect_track = effect.abs().max(dim=-1).values.max().item()
print(f"Maximum predicted regulatory effect: {max_effect_track:.3f}")

The Nucleotide Transformer — Scale and Transfer

The Nucleotide Transformer (Dalla-Torre et al., 2023) takes the DNA language model approach further, training models from 500 million to 2.5 billion parameters on genomes from 3,202 different species. The multi-species pretraining is the key innovation: evolutionary conservation across species is one of the strongest signals for functional importance, and a model trained on hundreds of species learns to recognise conserved functional elements that species-specific models miss.

The Nucleotide Transformer is available through the Hugging Face hub and can be used via the standard transformers API, making it one of the most accessible genomic foundation models for practical use.

from transformers import AutoTokenizer, AutoModelForMaskedLM
import torch

# Load Nucleotide Transformer (500M parameter version)
tokenizer = AutoTokenizer.from_pretrained(
    "InstaDeepAI/nucleotide-transformer-500m-multi-species"
)
nt_model = AutoModelForMaskedLM.from_pretrained(
    "InstaDeepAI/nucleotide-transformer-500m-multi-species"
)

# Embed a sequence
tokens  = tokenizer(["ATCGATCGATCG"], return_tensors='pt')
with torch.no_grad():
    output = nt_model(**tokens, output_hidden_states=True)
    # Use the last hidden state as embedding
    embedding = output.hidden_states[-1].mean(dim=1)
    print(f"Nucleotide Transformer embedding: {embedding.shape}")

Comparing the Models

Model Input length Best for Key limitation
DNABERT-2 Up to 512 tokens TF binding, promoters, splicing Short context only
Enformer 196,608 bp Gene expression prediction, variant effects Human/mouse only, complex setup
Nucleotide Transformer Up to 1,000 tokens Multi-species, conservation Medium context
AlphaGenome (Ch 14) Up to 1 Mb Full regulatory landscape, long-range Very compute-intensive
Reality Check
  • For short sequence classification tasks (promoter vs non-promoter, splice site), a CNN baseline often matches DNABERT with far less compute.
  • Enformer requires a GPU and substantial memory. For variant effect prediction on a personal computer, consider using the pre-computed Enformer scores available from public databases.
  • All these models were primarily trained and validated on human data. Performance on non-human species varies and should be validated before trusting predictions.
  • Does the task require long-range context? If your regulatory element is within 1,000 bp of the gene, DNABERT is sufficient. If it is 100,000 bp away, you need Enformer or AlphaGenome.
