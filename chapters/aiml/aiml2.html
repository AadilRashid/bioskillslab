Chapter 2: What is a Model?

~30 min Beginner Model, Features, Labels, Parameters, Training, Prediction

The Biological Question

You have gene expression measurements from 500 tumour samples. Each sample is either lung cancer or breast cancer. Can a computer learn to tell them apart — and then correctly classify a new sample it has never seen before?

This is the core task of machine learning in biology. Before writing a single line of code, you need to understand what a model actually is and how learning happens.

A Model is a Function

At its most fundamental level, a model is a function that maps inputs to outputs:

Input → Model → Output

Gene expression values → Model → Cancer subtype
DNA sequence          → Model → Regulatory element (yes/no)
Protein sequence      → Model → 3D structure
Cell gene expression  → Model → Cell type

The model is not magic. It is a mathematical function — a set of equations with adjustable numbers called parameters or weights. Learning means finding the right values for those parameters so the function produces correct outputs.

Three Things Every Model Needs

1. Features — The Input

Features are the measurements you feed into the model. In genomics, features are almost always numerical:

  • Gene expression values (e.g. TPM or raw counts for 20,000 genes)
  • Presence or absence of a mutation at each genomic position
  • Chromatin accessibility scores across the genome
  • Protein abundance measurements from proteomics

Choosing the right features matters enormously. A model can only learn from what you give it. If the signal you care about is not in your features, no amount of model complexity will find it.

2. Labels — The Answer You're Teaching

In supervised learning, every training example has a label — the correct answer the model is trying to learn to predict:

  • Cancer subtype (lung, breast, colon...)
  • Tumour vs normal
  • Patient survival: alive or deceased at 5 years
  • Regulatory element: active enhancer or not

Labels come from you — from clinical records, experimental assays, expert annotation. The quality of your labels determines the ceiling of what the model can learn. Garbage labels produce garbage models, no matter how sophisticated the algorithm.

3. A Learning Algorithm — How Parameters Are Updated

The learning algorithm adjusts the model's parameters to make predictions that match the labels. The process works like this:

1. Start with random parameter values
2. Make a prediction on a training example
3. Compare prediction to the true label
4. Calculate the error (the "loss")
5. Adjust parameters slightly to reduce the error
6. Repeat for thousands of examples
7. Keep adjusting until predictions are consistently good

This process is called training. Step 5 — adjusting parameters based on error — is called gradient descent. You do not need to understand the calculus behind it. You need to understand that learning = repeatedly adjusting parameters to reduce prediction error.

The weights analogy

Think of a model's parameters as dials on a mixing board. At the start, every dial is set randomly. Training turns each dial very slightly, millions of times, until the output sounds right. A deep neural network might have billions of dials. AlphaFold 2 has around 93 million parameters. GPT-4 has hundreds of billions.

What Does "Learning" Actually Mean?

Learning does not mean the model understands biology. It means the model has found a mathematical function that maps your features to your labels accurately on the training data.

Consider a simple example. You want to predict tumour vs normal from two gene expression values — Gene A and Gene B.

Sample   Gene_A   Gene_B   Label
──────   ──────   ──────   ─────
S1       8.2      1.1      Tumour
S2       1.3      7.8      Normal
S3       7.9      1.4      Tumour
S4       1.1      8.2      Normal
S5       8.5      0.9      Tumour

A model looks at hundreds of such examples and learns: "when Gene A is high and Gene B is low, it's probably a tumour." That pattern is captured as parameter values — numbers inside the model that encode the relationship between features and labels.

It has not "understood" the biology of why Gene A is elevated in tumours. It has learned a statistical association. This distinction matters when interpreting results.

Training vs Testing — The Most Important Distinction

A model that performs perfectly on the data it was trained on is not necessarily useful. It may have simply memorised the training examples rather than learning a generalisable pattern.

This is called overfitting — and it is the single most common failure mode in biological AI. (We dedicate a full chapter to it.)

To detect overfitting, you always split your data:

All data
├── Training set  (~70-80%)  ← Model learns from this
├── Validation set (~10-15%) ← Tune hyperparameters
└── Test set       (~10-15%) ← Final honest evaluation

The test set is held out completely until the very end. Its performance tells you how well the model will perform on genuinely new, unseen data — which is what actually matters in biology.

Data leakage

If test data accidentally influences training — through preprocessing, feature selection, or normalisation done on the full dataset before splitting — your evaluation is optimistic and your model is not as good as it looks. This happens more often than most papers admit. We return to this in Chapter 6.

Parameters vs Hyperparameters

Two terms that confuse almost every beginner:

Term What it is Who sets it
Parameters The weights inside the model — the numbers that get adjusted during training Learned automatically from data
Hyperparameters Settings that control how training works — learning rate, number of layers, tree depth Set by you before training

You choose hyperparameters. The model learns parameters. Choosing good hyperparameters is part of the art of machine learning — and part of why simply downloading a model and running it without understanding the settings produces unreliable results.

A Model is Not the Truth

George Box, a statistician, wrote: "All models are wrong, but some are useful." This is especially true in biology.

A model that predicts cancer subtype with 94% accuracy is not a biological truth. It is a useful approximation learned from a specific dataset, a specific feature set, a specific patient population, at a specific point in time. Change any of those and the accuracy may drop dramatically.

Every time you see a model applied to biological data, ask:

  • What data was it trained on?
  • Does my data come from the same distribution?
  • What could it have learned that is not biology — batch effects, cohort biases, technical artifacts?
Running Question progress

Our running question: Can we predict cancer subtype from gene expression?

After this chapter you now know what that question actually involves: features (gene expression values), labels (cancer subtypes), parameters (numbers learned from the data), and a test set (to honestly evaluate whether it works). In Chapter 7 we start building it.

Reality Check
  • A model with high training accuracy may have simply memorised the data
  • The test set performance is the only number that matters
  • A model is only as good as the labels it was trained on
  • Could you answer the same question with a simpler statistical test?
