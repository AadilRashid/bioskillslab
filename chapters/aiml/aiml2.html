Chapter 2: What is a Model?

~20 min Beginner Model, Features, Labels, Parameters, Training, Overfitting

The Biological Question

You have gene expression measurements from 500 tumour samples — some lung cancer, some breast cancer. Can a computer learn to tell them apart, and correctly classify a new sample it has never seen?

This is the core task of machine learning in biology. Before writing a single line of code, you need to understand what a model is.

A Model is a Function

A model takes inputs and produces outputs. In genomics:

The model is not magic. It is a mathematical function with adjustable numbers called parameters or weights. Learning means finding the right values for those parameters so the function produces correct outputs.

Three Things Every Model Needs

Features

The measurements you feed in. Gene expression values, mutation presence/absence, chromatin accessibility. A model can only learn from what you give it.

Labels

The correct answers for each training example. Cancer subtype, tumour vs normal, patient outcome. Labels come from clinical records, experimental assays, or expert annotation.

Learning Algorithm

The process that adjusts parameters to reduce prediction error. Make prediction, calculate error, adjust weights slightly, repeat thousands of times. This is training.

The weights analogy

Think of parameters as dials on a mixing board. Training turns each dial very slightly, millions of times, until the output is right. AlphaFold 2 has 93 million dials. GPT-4 has hundreds of billions.

Training vs Testing

A model that performs perfectly on training data may have simply memorised it rather than learned anything generalisable. This is overfitting — the most common failure mode in biological AI.

You always split your data before training:

All data
├── Training set   (~70-80%)  Model learns from this
├── Validation set (~10-15%)  Tune settings
└── Test set       (~10-15%)  Final honest evaluation

The test set is held out completely until the end. Its performance is the only number that honestly tells you whether the model will work on new data.

Parameters vs Hyperparameters

Term What it is Who sets it
Parameters The weights inside the model — numbers adjusted during training Learned from data
Hyperparameters Settings that control how training works — learning rate, number of layers Set by you before training

A Model is Not the Truth

George Box wrote: "All models are wrong, but some are useful." A model predicting cancer subtype with 94% accuracy is not biological truth. It is an approximation learned from a specific dataset at a specific time. Change the dataset and the accuracy may drop dramatically.

Reality Check
  • High training accuracy may mean memorisation, not learning
  • Test set performance is the only number that matters
  • A model is only as good as its labels
  • Could a simpler statistical test answer the same question?

Running Question

Can we predict cancer subtype from gene expression? You now know what that involves: features (expression values), labels (subtypes), parameters (learned numbers), and a test set. In Chapter 7 we start building it.
