Three Frontiers

This chapter covers three areas that sit at the current frontier of biological AI: long-range regulatory genomics with AlphaGenome, integrating multiple molecular data types simultaneously, and reasoning over biological networks with graph neural networks. Each represents a genuinely different kind of biological question that requires genuinely different computational approaches.

AlphaGenome — The Full Regulatory Landscape

AlphaGenome (Google DeepMind, Nature 2026) is the successor to Enformer and represents a fundamental advance in what is possible from DNA sequence alone. Where Enformer takes ~200 kilobases of sequence and predicts a fixed set of genomic tracks, AlphaGenome takes up to 1 megabase of DNA and predicts a much broader range of regulatory outputs simultaneously from a single unified model.

The tasks AlphaGenome predicts include gene expression across tissues, chromatin accessibility, histone modifications, transcription factor binding, splicing, chromatin contacts (3D genome organisation), and variant effects. This multimodal prediction from a single model is the key architectural advance. Earlier approaches required separate models for each regulatory modality — one for expression, one for chromatin, one for splicing. AlphaGenome learns the relationships between these modalities jointly, which allows it to make more accurate predictions for each.

AlphaGenome vs Enformer

Property Enformer AlphaGenome
Context window ~200 kb Up to 1 Mb
Output modalities Expression, chromatin tracks Expression, chromatin, splicing, 3D contacts, variants
Variant benchmarks Strong baseline Best on 25 of 26 variant-effect benchmarks
Architecture Conv + Transformer Unified multimodal transformer
Availability Open weights DeepMind tooling released alongside paper

The performance improvement on variant-effect prediction benchmarks is the most clinically relevant advance. Predicting which non-coding variants affect gene regulation is one of the hardest problems in human genetics — most variants identified in GWAS studies are non-coding, and understanding their functional consequences requires precisely the kind of long-range regulatory modelling that AlphaGenome was designed for.

# AlphaGenome usage (DeepMind Python API)
# Installation and weights from DeepMind's release

from alphagenome import AlphaGenome, one_hot_encode
import numpy as np

# Load pretrained model
model = AlphaGenome.from_pretrained()

# Predict regulatory outputs from a 1 Mb sequence
sequence = "ATCG..."  # 1,000,000 characters
x = one_hot_encode(sequence)

predictions = model.predict(x)
# Returns dict with keys:
# 'gene_expression': tissue-specific expression
# 'chromatin': accessibility tracks
# 'splicing': splice site usage
# 'contacts': 3D chromatin contacts
# 'variant_effects': ISM scores

# Variant effect prediction (in silico mutagenesis)
variant_effects = model.predict_variant_effect(
    sequence=sequence,
    position=500000,  # variant position within the sequence
    ref='A',
    alt='G'
)
print("Predicted regulatory effect:")
print(variant_effects)
When to use AlphaGenome vs Enformer

For most research tasks today, Enformer remains practical and well-documented with an established community. AlphaGenome is the right choice when you need the most accurate variant effect predictions, when you need to predict splicing alongside expression, or when your regulatory element is beyond Enformer's 200 kb window. For learning and exploration, start with Enformer — the concepts transfer directly to AlphaGenome.

Multi-Omics AI — Integrating Multiple Data Types

Gene expression is one view of a cell's state. Chromatin accessibility shows which regions of the genome are accessible to transcription factors. Protein abundance reflects post-transcriptional regulation. Methylation captures epigenetic memory. Each modality provides a different, partially overlapping picture of what is happening biologically.

Multi-omics AI methods learn joint representations that capture the information from all modalities simultaneously. The key insight is that the correlations between modalities — how chromatin accessibility predicts transcription, how transcription predicts protein levels — contain biological information that no single modality can provide alone. The three frameworks you should know are MOFA+, totalVI, and MultiVI, in order of conceptual progression.

MOFA+ — Multi-Omics Factor Analysis

MOFA+ (Multi-Omics Factor Analysis, Argelaguet et al., 2020) is the conceptual starting point for multi-omics integration. Think of it as PCA extended to multiple data types simultaneously. It learns a set of latent factors, each of which explains correlated variation across all measured modalities.

A factor might capture variation that is visible in both RNA-seq and ATAC-seq data — regulatory activity at a set of loci that drives transcription of a set of genes. Another factor might capture variation visible only in RNA-seq — perhaps post-transcriptional regulation that does not leave an ATAC-seq signature. MOFA+ can distinguish these.

from mofapy2.run.entry_point import entry_point
import pandas as pd

# MOFA+ expects a list of data matrices, one per modality
# Each matrix: samples x features

data = [
    rna_matrix,    # samples x genes
    atac_matrix,   # samples x peaks
    protein_matrix # samples x proteins
]
views = ['RNA', 'ATAC', 'Protein']

# Initialise and train MOFA+
ent = entry_point()
ent.set_data_options(scale_groups=False, scale_views=False)
ent.set_data_df(
    pd.concat([pd.DataFrame(d, columns=[f'{v}_{i}' for i in range(d.shape[1])])
               for d, v in zip(data, views)], axis=1)
)
ent.set_model_options(factors=15)
ent.set_train_options(iter=1000, convergence_mode='fast')
ent.build()
ent.run()

# Extract factor scores per sample
factors = ent.model.getExpectations()['Z']['E']
print(f"Factor scores shape: {factors.shape}")  # (n_samples, 15)

# Which features load onto each factor?
weights = ent.model.getExpectations()['W']
for view, w in zip(views, weights):
    print(f"Top features for Factor 1 in {view}:")
    top_features = pd.Series(w['E'][:, 0]).abs().nlargest(10)
    print(top_features)

MOFA+ is the right choice when biological interpretation matters more than predictive performance. The factors are interpretable — you can examine what they capture and make biological claims about what is driving the variation in your data. For clinical cohorts with matched multi-omics, MOFA+ factors that separate patient groups often correspond to known disease mechanisms.

totalVI — RNA and Protein Together

totalVI (Gayoso et al., Nature Methods 2021) extends the scVI framework from Chapter 11 to CITE-seq data — experiments that simultaneously measure RNA expression and surface protein levels in single cells. The two modalities have different statistical properties: RNA follows a negative binomial distribution, while protein counts from antibody-derived tags (ADT) have different noise characteristics.

totalVI learns a joint latent space that captures the shared and modality-specific variation, handles batch effects across both modalities simultaneously, and enables protein expression imputation for cells where protein data is missing.

import scvi

# Setup CITE-seq data (RNA + protein)
scvi.model.TOTALVI.setup_anndata(
    adata,
    layer='counts',           # raw RNA counts
    protein_expression_obsm_key='protein_expression',  # raw protein counts
    batch_key='batch'
)

model = scvi.model.TOTALVI(adata, latent_distribution='ln')
model.train(max_epochs=400)

# Get joint latent representation
adata.obsm['X_totalVI'] = model.get_latent_representation()

# Impute protein expression for all cells
# (useful when protein data is missing or noisy)
protein_imputed = model.get_protein_foreground_probability()
print(f"Imputed protein shape: {protein_imputed.shape}")

MultiVI — RNA and ATAC Together

MultiVI (Ashuach et al., Nature Methods 2023) extends scVI to paired single-cell multi-omics data — specifically the 10x Multiome protocol that simultaneously measures gene expression and chromatin accessibility in the same cell. It is the most directly relevant multi-omics tool for students working with modern single-cell data.

MultiVI learns a joint latent representation across RNA and ATAC modalities, enables imputation of missing modalities, and can be used for batch integration across experiments that measured different combinations of modalities — some measured RNA only, some ATAC only, some both.

import scvi

# Setup 10x Multiome data
scvi.model.MULTIVI.setup_anndata(
    adata,
    layer='counts',                     # RNA raw counts
    atac_key='atac_counts',             # ATAC raw counts
    batch_key='batch'
)

model = scvi.model.MULTIVI(
    adata,
    n_genes=adata.n_vars,
    n_regions=adata.obsm['atac_counts'].shape[1]
)
model.train(max_epochs=500)

# Joint embedding across both modalities
adata.obsm['X_MultiVI'] = model.get_latent_representation()

# Impute ATAC for RNA-only cells
atac_imputed = model.get_accessibility_estimates()

The progression from MOFA+ to totalVI to MultiVI mirrors the conceptual journey from classical factor analysis to deep generative models. MOFA+ is interpretable and parameter-efficient, ideal for small cohorts. totalVI and MultiVI are more powerful for large single-cell datasets but require more data and compute to train reliably.

Graph Neural Networks for Biological Networks

Everything covered so far has treated biological data as matrices — samples by features, cells by genes. But much of biology is fundamentally relational: proteins interact with other proteins, genes regulate other genes, metabolites feed into pathways that feed into other pathways. Networks are the natural representation for this kind of data, and graph neural networks (GNNs) are the natural tool for learning from them.

The Biological Graphs You Will Encounter

Two types of biological networks are most commonly used in computational biology.

Protein-protein interaction (PPI) networks. The STRING database curates interactions between proteins across all sequenced organisms, integrating experimental evidence, co-expression data, and text mining. Nodes are proteins; edges represent known or predicted interactions weighted by confidence. A GNN trained on a PPI network can predict the function of an uncharacterised protein from the functions of its interaction partners — guilt by association at scale.

Gene regulatory networks (GRNs). Nodes are genes; directed edges represent regulatory relationships — TP53 activates CDKN1A, MYC activates CCND1, and so on. GNNs can learn to predict which genes a transcription factor regulates from the network topology and sequence features, or to predict the downstream effects of knocking out a transcription factor.

PyTorch Geometric — The Standard Framework

PyTorch Geometric (PyG) has become the dominant framework for biological GNNs. It integrates cleanly with PyTorch and provides implementations of all major GNN architectures.

Graph Convolutional Network for Protein Function Prediction

class ProteinGCN(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels):
        super().__init__()
        self.conv1 = GCNConv(in_channels, hidden_channels)
        self.conv2 = GCNConv(hidden_channels, hidden_channels)
        self.conv3 = GCNConv(hidden_channels, out_channels)

    def forward(self, x, edge_index):
        # Layer 1: aggregate information from 1-hop neighbours
        x = self.conv1(x, edge_index)
        x = F.relu(x)
        x = F.dropout(x, p=0.3, training=self.training)

        # Layer 2: aggregate from 2-hop neighbours
        x = self.conv2(x, edge_index)
        x = F.relu(x)
        x = F.dropout(x, p=0.3, training=self.training)

        # Layer 3: output layer
        x = self.conv3(x, edge_index)
        return x

Graph Attention Networks — Learning Which Neighbours Matter

A standard GCN treats all neighbours equally when aggregating information. Graph Attention Networks (GAT) learn attention weights — some interaction partners are more informative than others for a given prediction task, and the model learns which ones to attend to. In biological networks this is biologically sensible: a protein's strongest interaction partners are usually more informative about its function than its weak or indirect connections.

from torch_geometric.nn import GATConv

class ProteinGAT(torch.nn.Module):
    def __init__(self, in_channels, hidden_channels, out_channels, heads=4):
        super().__init__()
        self.gat1 = GATConv(in_channels, hidden_channels, heads=heads)
        self.gat2 = GATConv(hidden_channels * heads, out_channels, heads=1)

    def forward(self, x, edge_index):
        x = self.gat1(x, edge_index)
        x = F.elu(x)
        x = self.gat2(x, edge_index)
        return x

Connecting Everything

This chapter covers three seemingly different areas — long-range DNA regulation, multi-omics integration, and biological network analysis — but they are connected by a common theme: biology is not modular. Gene regulation (AlphaGenome) affects what gets transcribed. Transcription, chromatin, and protein levels are jointly regulated (multi-omics). Proteins interact in networks that determine cellular function (GNNs). A complete picture of a cell or a disease requires all three levels of understanding, and the AI tools covered in this chapter provide the means to analyse each level.

Reality Check
  • For multi-omics: does your dataset actually have matched measurements across modalities? MOFA+ can handle partially overlapping data, but MultiVI and totalVI work best with paired data from the same cells.
  • For GNNs: is the network you are using complete and reliable? STRING confidence scores matter — low-confidence edges add noise.
  • For AlphaGenome: does your variant lie within the 1 Mb context window? Variants outside this window will not be modelled accurately.
  • Always compare GNN performance to a simple baseline that uses only node features without the graph structure. If the graph adds no value, you may have the wrong network or the wrong features.
