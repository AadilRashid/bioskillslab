The Dataset: TCGA Pan-Cancer

The Cancer Genome Atlas is a landmark programme that systematically collected and analysed tumour samples from thousands of patients across 33 cancer types. It produced matched genomic, transcriptomic, proteomic, and clinical data — all publicly available.

For this course we use the RNA-seq gene expression data from the Pan-Cancer Atlas: roughly 10,000 tumour samples, each with expression measurements for around 20,000 genes, spanning 33 cancer types. This is the dataset we will use from Chapter 7 through Chapter 10 to build and compare increasingly sophisticated classifiers.

Dataset:    TCGA Pan-Cancer Atlas (RNA-seq)

The Gene Expression Matrix

Every machine learning task in genomics starts with the same fundamental data structure: the expression matrix. Understanding this matrix deeply is more important than understanding any algorithm.

The expression matrix is a table where rows are genes and columns are samples (or vice versa, depending on convention). Each cell contains the measured expression level of that gene in that sample.

         Sample_1  Sample_2  Sample_3  Sample_4  ...
GENE_A     12.4      0.3       11.8      0.1     ...
GENE_B      0.1      8.7        0.2      9.1     ...
GENE_C      5.6      5.2        4.9      5.8     ...
BRCA1       3.2      0.8        4.1      0.5     ...
TP53        7.8      6.9        8.2      7.1     ...
...         ...      ...        ...      ...

Sample labels:
  Sample_1: BRCA (breast cancer)
  Sample_2: LUAD (lung adenocarcinoma)
  Sample_3: BRCA
  Sample_4: LUAD

For machine learning, you transpose this so that rows are samples and columns are features (genes). Each sample is one data point described by 20,000 feature values.

import pandas as pd
import numpy as np

# Load expression matrix
expr = pd.read_csv('tcga_pancan_expression.tsv', sep='\t', index_col=0)

# Transpose: rows = samples, columns = genes
expr = expr.T

print(f"Samples: {expr.shape[0]}")   # ~10,000
print(f"Genes:   {expr.shape[1]}")   # ~20,000

# Load labels
labels = pd.read_csv('tcga_labels.tsv', sep='\t', index_col=0)
print(labels['cancer_type'].value_counts())

Why Raw Counts Need Transformation

RNA-seq counts are not directly comparable across samples. Two samples sequenced to different depths — one producing 20 million reads and another 40 million — will show different raw counts for the same gene purely because of sequencing depth, not biology.

Normalisation removes these technical differences. For gene expression data used in machine learning, two transformations are almost always applied:

Step 1: Log transformation. Gene expression values span several orders of magnitude. A highly expressed housekeeping gene might have counts of 100,000 while a lowly expressed transcription factor might have counts of 10. This enormous range causes algorithms to weight highly expressed genes far more heavily than they deserve. Log transformation compresses the range:

# Log2 transform (add 1 to avoid log(0))
expr_log = np.log2(expr + 1)

# Before: range 0 to 100,000
# After:  range 0 to ~17

Step 2: Standardisation (z-score scaling). After log transformation, different genes still have different means and variances. Standardisation puts every gene on the same scale — zero mean, unit variance — so that no gene dominates the analysis simply because it happens to have higher absolute values.

from sklearn.preprocessing import StandardScaler

# Fit scaler on TRAINING data only
# Never fit on the full dataset including test samples
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)

# Apply the SAME scaler to test data
X_test_scaled = scaler.transform(X_test)  # note: transform, not fit_transform
Fit normalisation on training data only

This is the leakage point we discussed in Chapter 6. If you call fit_transform on the entire dataset before splitting, the test set's statistics influenced your normalisation parameters. The scaler must be fitted on training data only, then applied to the test set using the training statistics. This is not optional — it is fundamental to honest evaluation.

PCA on TCGA — Before Any Model

Before training a single classifier, we run PCA. This is not optional — it is the first thing you do with any high-dimensional genomics dataset. PCA tells you whether the signal you care about actually exists and is separable in your data.

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
import matplotlib.pyplot as plt
import pandas as pd

# Assume X is your expression matrix (samples x genes), y is cancer type labels

# Step 1: Scale
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Step 2: PCA
pca = PCA(n_components=50)
X_pca = pca.fit_transform(X_scaled)

# Variance explained by each component
variance_explained = pca.explained_variance_ratio_
print(f"PC1 explains {variance_explained[0]*100:.1f}% of variance")
print(f"PC2 explains {variance_explained[1]*100:.1f}% of variance")
print(f"Top 10 PCs explain {sum(variance_explained[:10])*100:.1f}% of variance")

# Step 3: Visualise PC1 vs PC2
plt.figure(figsize=(10, 8))
cancer_types = y.unique()
for ct in cancer_types:
    mask = y == ct
    plt.scatter(X_pca[mask, 0], X_pca[mask, 1], label=ct, alpha=0.5, s=10)

plt.xlabel(f'PC1 ({variance_explained[0]*100:.1f}%)')
plt.ylabel(f'PC2 ({variance_explained[1]*100:.1f}%)')
plt.title('PCA of TCGA Pan-Cancer Expression')
plt.legend(bbox_to_anchor=(1.05, 1), loc='upper left', fontsize=6)
plt.tight_layout()
plt.show()

When you run this on TCGA data, you will see something remarkable: many cancer types separate clearly in the first two principal components, using only two numbers derived from 20,000 gene measurements. Breast cancer clusters together. Kidney cancers cluster together. Brain tumours cluster together. The biology is visible in the data before any supervised learning has been done.

This is an important finding in itself. It tells you that a simple linear classifier — logistic regression on PCA components — will likely work well. And it sets up the key question of Chapter 8: how much better, if at all, does a more complex model do?

Elbow Plot — How Many Components?

The scree plot (elbow plot) shows how much variance each principal component explains. You look for the "elbow" — the point where adding more components stops contributing meaningfully.

# Fit PCA with more components to see the elbow
pca_full = PCA(n_components=100)
pca_full.fit(X_scaled)

plt.figure(figsize=(10, 4))
plt.plot(range(1, 101), pca_full.explained_variance_ratio_ * 100, 'o-', markersize=3)
plt.xlabel('Principal Component')
plt.ylabel('Variance Explained (%)')
plt.title('Scree Plot — TCGA Pan-Cancer')
plt.axvline(x=20, color='red', linestyle='--', label='Elbow ~PC20')
plt.legend()
plt.show()

# Cumulative variance
cumulative = np.cumsum(pca_full.explained_variance_ratio_)
print(f"10 PCs explain: {cumulative[9]*100:.1f}%")
print(f"20 PCs explain: {cumulative[19]*100:.1f}%")
print(f"50 PCs explain: {cumulative[49]*100:.1f}%")

For TCGA data, the elbow typically appears around 20-30 components, which together explain 60-70% of the total variance. Using 50 components for downstream classification captures the vast majority of the signal while discarding most of the noise.

Checking for Batch Effects

Before trusting your PCA, check whether samples are separating by biology or by technical factors. Colour your PCA plot by sequencing batch, collection site, or year of sample collection. If samples cluster by any of these, you have a batch effect that must be corrected before training any model.

# Colour by batch instead of cancer type
plt.figure(figsize=(10, 8))
for batch in metadata['batch'].unique():
    mask = metadata['batch'] == batch
    plt.scatter(X_pca[mask, 0], X_pca[mask, 1],
                label=f'Batch {batch}', alpha=0.5, s=10)

plt.xlabel('PC1')
plt.ylabel('PC2')
plt.title('PCA coloured by sequencing batch')
plt.legend()
plt.show()

# If samples cluster by batch: apply batch correction before proceeding
# Options: ComBat (sva package in R), limma::removeBatchEffect,
#          or include batch as a covariate in the model
TCGA and batch effects

TCGA data was collected across many institutions over many years. The Pan-Cancer Atlas team invested substantial effort in batch correction before releasing the data. For the pre-processed TCGA expression file from UCSC Xena, major batch effects have already been addressed. For your own data, you should always check.

Preparing the Final Feature Matrix

After PCA exploration, you are ready to prepare the data for supervised classification. The standard pipeline:

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, LabelEncoder
from sklearn.decomposition import PCA

# Encode string labels to integers
le = LabelEncoder()
y_encoded = le.fit_transform(y)  # 'BRCA' -> 0, 'LUAD' -> 1, etc.

# Split at sample level (no patient overlap issues in TCGA bulk data)
X_train, X_test, y_train, y_test = train_test_split(
    X, y_encoded, test_size=0.2, random_state=42, stratify=y_encoded
)
# stratify=y_encoded ensures each cancer type is proportionally represented

# Scale: fit on train only
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled  = scaler.transform(X_test)

# PCA: fit on train only
pca = PCA(n_components=50)
X_train_pca = pca.fit_transform(X_train_scaled)
X_test_pca  = pca.transform(X_test_scaled)

print(f"Training samples: {X_train_pca.shape[0]}")
print(f"Test samples:     {X_test_pca.shape[0]}")
print(f"Features (PCs):   {X_train_pca.shape[1]}")

You now have a clean, normalised, dimensionality-reduced feature matrix ready for classification. In Chapter 8 we train the first models on this data and get our initial answer to the running question.

Reality Check
  • Does PCA separate cancer types? If not, the classifier will struggle regardless of complexity.
  • Did you fit all preprocessing (scaler, PCA) on training data only?
  • Did you use stratified splitting to ensure all cancer types appear in both train and test?
  • Have you checked for batch effects before claiming any biological signal?
