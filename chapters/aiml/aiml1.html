Chapter 1: AI, ML, Deep Learning, LLMs — The Mental Map

~35 min Beginner AI, ML, Deep Learning, LLMs, Foundation Models

Two Things That Are Both True

AI has predicted protein structures that would have taken years of experimental work to determine. AlphaFold2 solved a problem that structural biologists had worked on for half a century — in months.

AI has also "discovered" biological patterns that later turned out to be technical artifacts, batch effects, or data leakage. Papers have been published. Some have been retracted.

The one skill this course teaches

Understanding the difference between those two outcomes — when AI genuinely advances biology and when it produces convincing nonsense — is the most important skill in modern computational biology. Everything in this course builds toward that judgement.

Why Biology Suddenly Needs AI

Biology has changed. Not gradually — suddenly.

A decade ago, a typical experiment produced hundreds of measurements. Today a single RNA-seq run produces expression values for 20,000 genes across hundreds of samples. A single-cell experiment measures millions of individual cells. A spatial transcriptomics slide captures gene expression at every position across a tissue slice.

The data is no longer the bottleneck. Making sense of it is.

Human intuition, spreadsheets, and t-tests were designed for a world where data was scarce. They were not designed for:

  • 20,000 genes measured simultaneously
  • Millions of cells profiled in a single experiment
  • 3 billion base pairs of DNA sequence per patient
  • Multi-omics data combining genomics, transcriptomics, proteomics, and clinical records

This is precisely the kind of problem AI was built for — finding patterns in high-dimensional data that humans cannot inspect manually.

What is Artificial Intelligence?

AI is the broader goal of making computers perform tasks that normally require human intelligence — recognising images, understanding language, making decisions, discovering patterns.

In biology, AI tasks include:

  • Identifying cell types from gene expression profiles
  • Predicting whether a mutation causes disease
  • Determining protein structure from amino acid sequence
  • Discovering patient subgroups from multi-omics data

What is Machine Learning?

Instead of writing explicit rules, you show the computer many examples and let it learn the rules itself.

Traditional programming:   Rules + Data → Output
Machine learning:          Data + Output → Rules (learned automatically)

A machine learning model for cancer subtyping does not need a biologist to specify which genes matter. It learns which combinations of gene expression values co-occur with which cancer subtypes — from thousands of patient samples.

This is powerful. It is also dangerous. The model learns whatever patterns exist in the data, including batch effects, population biases, and technical artifacts. More on this in Chapter 6.

What is Deep Learning?

Standard machine learning works well when you know which features matter. Deep learning solves the case when you don't — by learning useful representations automatically from raw data.

Standard ML:    You select features → Model learns from features
Deep learning:  Raw data → Network learns features → Network learns predictions

In genomics, this matters because 20,000 genes is too many to manually curate. Deep networks can compress those 20,000 measurements into 50 meaningful numbers — this is what autoencoders do (Chapter 9).

What is a Transformer?

Transformers are neural networks designed for sequences and long-range relationships. The key innovation is attention: the ability to look at any part of a sequence when interpreting any other part.

This matters enormously for DNA, where a regulatory element 500,000 base pairs away can control a gene's expression. Earlier models could not capture relationships at that scale. Transformers can.

What is a Large Language Model?

LLMs are transformers trained on enormous datasets. GPT-4 is trained on text. But the same architecture trained on DNA sequences produces a DNA language model. Trained on protein sequences, a protein language model.

In biology, LLMs assist with literature summarisation, protocol generation, and bioinformatics copilots. We cover these in Chapter 15. For now, understand that LLMs are one application of transformer architecture — not the only one, and not always the most relevant one for genomics.

What is a Foundation Model?

A foundation model is pretrained on enormous biological datasets and can be adapted to many downstream tasks. In 2026, the most important biological foundation models are:

  • Geneformer — pretrained on 30 million single-cell transcriptomes
  • scGPT — GPT-style model for single-cell biology
  • Nucleotide Transformer — DNA sequence foundation model
  • AlphaGenome — predicts gene expression, chromatin accessibility, and variant effects from up to 1 Mb of DNA
  • ESM3 — multimodal protein foundation model
  • AlphaFold 3 — extends structure prediction to protein-DNA-RNA-ligand interactions

You will work with these in the second half of this course.

What is an Agent?

An agent is a foundation model that can use tools, retain memory, and take actions. In biology, agents are beginning to automate literature review, design experiments, and run analysis pipelines. We cover this briefly in Chapter 15 — the core concepts of foundation models are far more important right now.

Reality Check — every chapter asks this
  • Does AI actually outperform a simpler method here?
  • Is the dataset large enough to trust these results?
  • Could PCA or a random forest do this just as well?

The Running Question

Can we predict cancer subtype from gene expression?

And — was the complexity of AI actually necessary to do it?

By the capstone you will have answered this using tools that span from PCA to foundation models — and you will know which one genuinely helped.
