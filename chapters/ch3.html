~75 min Beginner Shell, Pipes, grep, awk, sed, cut, sort

Why Unix?

Almost every bioinformatics tool — samtools, BWA, BLAST, BEDTools, GATK — is designed to run on Unix/Linux. Servers, HPC clusters, and cloud instances all run Linux. The command line isn't optional in bioinformatics; it's the primary interface.

The Unix philosophy is simple but powerful: write small programs that do one thing well, and chain them together with pipes. This lets you build complex data processing workflows from simple building blocks.

Navigating the Filesystem

The Linux filesystem is a tree. Everything starts at the root /. Your home directory is /home/username (shortcut: ~).

# Where am I right now?
pwd                          # /home/user/project

# What's in this directory?
ls                           # Simple list
ls -l                        # Long format (permissions, size, date)
ls -lah                      # + hidden files + human-readable sizes
ls -lhS                      # Sort by size (largest first)
ls -lt                       # Sort by modification time (newest first)

# Move around
cd /path/to/dir              # Go to absolute path
cd project/data              # Go to relative path
cd ..                        # Go up one level
cd ../..                     # Go up two levels
cd ~                         # Go home
cd -                         # Go to previous directory (toggle)
Tab Completion Press Tab to auto-complete file and directory names. Press Tab twice to see all options. This saves enormous time and prevents typos.

Working with Files

# View file contents
cat file.txt                 # Print entire file (careful with large files!)
head -n 20 file.txt          # First 20 lines
tail -n 20 file.txt          # Last 20 lines
tail -n +2 file.txt          # Everything EXCEPT the first line (skip header)
less file.txt                # Scroll through file (q to quit, / to search)

# File info
wc -l file.txt               # Count lines
wc -w file.txt               # Count words
wc -c file.txt               # Count bytes
file sequences.fasta         # Detect file type (ASCII text, gzip, etc.)
du -sh data/                 # Directory size (human-readable)

# Copy, move, delete
cp source.txt dest.txt       # Copy file
cp -r source_dir/ dest_dir/  # Copy directory recursively
mv old_name.txt new_name.txt # Rename
mv file.txt other_dir/       # Move to another directory
rm file.txt                  # Delete (NO UNDO — no recycle bin!)
rm -r directory/             # Delete directory and contents
rm -i file.txt               # Ask for confirmation before deleting

# Create
mkdir new_dir                # Create directory
mkdir -p a/b/c/d             # Create nested directories
touch new_file.txt           # Create empty file (or update timestamp)
rm is permanent There is no recycle bin on the command line. rm -rf / would destroy your entire system. Always double-check before deleting. Consider using rm -i for interactive confirmation.

Redirection & Pipes — The Core of Unix Power

Every Unix command has three data streams:

stdin  (0) ← input  (keyboard or piped data)
stdout (1) → output (screen or redirected to file)
stderr (2) → errors (screen or redirected to file)

Redirection

# Redirect stdout to a file
echo "hello" > out.txt           # Overwrite (creates or replaces)
echo "world" >> out.txt          # Append to existing file

# Redirect stderr
command 2> errors.log            # Errors go to file
command > out.txt 2>&1           # Both stdout AND stderr to same file
command > /dev/null 2>&1         # Discard all output (silent mode)

Pipes — Chaining Commands

The pipe | sends the stdout of one command as stdin to the next. This is the most powerful concept in Unix:

# Count sequences in a FASTA file
cat sequences.fasta | grep ">" | wc -l

# Same thing, more efficiently (no need for cat)
grep -c ">" sequences.fasta

# Find the top 5 most common chromosomes in a BED file
cut -f1 genes.bed | sort | uniq -c | sort -rn | head -5

# Get average read length from FASTQ
awk 'NR%4==2{sum+=length($0);c++}END{print sum/c}' reads.fastq
Useless Use of Cat cat file | grep pattern is redundant — grep pattern file does the same thing. But cat is fine when you need to combine multiple files: cat file1 file2 | grep pattern.

grep — Pattern Searching

grep is your most-used bioinformatics command. It searches for text patterns in files.

# Basic search
grep "BRCA1" genes.bed                # Lines containing BRCA1
grep -i "homo sapiens" seqs.fa        # Case-insensitive
grep -w "chr1" genes.bed              # Whole word only (not chr10, chr11)
grep -n "ERROR" logfile.txt           # Show line numbers

# Counting
grep -c ">" sequences.fasta           # Count FASTA sequences
grep -c "^@" reads.fastq              # Count FASTQ reads (careful: @ also in quality)

# Inverting
grep -v "^#" variants.vcf             # Exclude comment/header lines
grep -v "^$" file.txt                 # Remove empty lines

# Context
grep -A 1 ">" sequences.fasta        # Show 1 line After each match (header + sequence)
grep -B 2 "ERROR" log.txt            # Show 2 lines Before each match

# Regular expressions
grep "^chr[0-9]" genes.bed           # Lines starting with chr followed by a digit
grep -E "chr(1|2|X)" genes.bed       # Extended regex: chr1, chr2, or chrX
grep -P "\t" file.txt                # Perl regex: match literal tab

awk — Column Processing Powerhouse

awk treats each line as a record with fields (columns) separated by whitespace. It's incredibly powerful for tabular bioinformatics data.

# Basic syntax: awk 'condition{action}' file

# Print specific columns
awk '{print $1, $4}' genes.bed           # Chromosome and gene name
awk '{print $1"\t"$4}' genes.bed         # Tab-separated output

# Filter by column value
awk '$5 > 900' genes.bed                 # Genes with score > 900
awk '$1 == "chr1"' genes.bed             # Only chromosome 1
awk '$6 == "+"' genes.bed                # Only forward strand genes

# Calculations
awk '{print $4, $3-$2}' genes.bed        # Gene name and length (end - start)
awk '{sum+=$5} END{print sum/NR}' genes.bed  # Average score

# FASTQ: every 4th line starting from line 2 = sequence
awk 'NR%4==2' reads.fastq                # Extract all sequences
awk 'NR%4==2{print length($0)}' reads.fastq  # Read lengths
awk 'NR%4==2{sum+=length($0);c++}END{print "Avg:",sum/c}' reads.fastq

# FASTA: calculate sequence lengths
awk '/^>/{if(seq)print name,length(seq); name=$0; seq=""} 
     !/^>/{seq=seq$0} 
     END{print name,length(seq)}' sequences.fasta

# Custom field separator
awk -F'\t' '{print $1, $2}' data.tsv     # Tab-separated input
awk -F',' '{print $1, $3}' data.csv      # Comma-separated input

# Built-in variables
# NR = current line number, NF = number of fields, $0 = entire line
awk 'NR==1' file.txt                     # First line only (header)
awk 'NR>1' file.txt                      # Skip header
awk '{print NF}' file.txt               # Count columns per line

cut — Extract Columns

cut -f1,4 genes.bed              # Columns 1 and 4 (tab-delimited by default)
cut -f1-3 genes.bed              # Columns 1 through 3
cut -f4 genes.bed                # Column 4 only
cut -d',' -f2 data.csv           # Column 2, comma-delimited
cut -c1-10 file.txt              # First 10 characters of each line

sort & uniq — Ordering and Counting

# Sort
sort file.txt                    # Alphabetical
sort -n file.txt                 # Numeric
sort -r file.txt                 # Reverse
sort -k5,5n genes.bed            # Sort by column 5, numeric
sort -k5,5nr genes.bed           # Sort by column 5, numeric, descending
sort -k1,1 -k2,2n genes.bed     # Sort by col 1 alpha, then col 2 numeric

# Unique (REQUIRES sorted input)
sort file.txt | uniq             # Remove adjacent duplicates
sort file.txt | uniq -c          # Count occurrences
sort file.txt | uniq -d          # Show only duplicates

# Common bioinformatics pattern: count items per category
cut -f1 genes.bed | sort | uniq -c | sort -rn
# Output:
#   3 chr1
#   2 chr2
#   1 chr3

sed — Stream Editor

sed 's/chr//' genes.bed              # Remove "chr" prefix from each line
sed 's/\t/,/g' data.tsv             # Convert TSV to CSV
sed -n '10,20p' file.txt            # Print lines 10-20
sed '/^#/d' file.vcf                # Delete comment lines
sed '1d' file.txt                   # Delete first line (header)
sed 's/old/new/' file.txt           # Replace first occurrence per line
sed 's/old/new/g' file.txt          # Replace ALL occurrences per line

Putting It All Together — Real Bioinformatics One-Liners

# 1. Count sequences in a FASTA file
grep -c "^>" sequences.fasta

# 2. Get all sequence IDs from a FASTA
grep "^>" sequences.fasta | sed 's/>//' | cut -d' ' -f1

# 3. Calculate average read length from FASTQ
awk 'NR%4==2{sum+=length($0);c++}END{printf "%.1f\n",sum/c}' reads.fastq

# 4. Find the longest sequence in a FASTA
awk '/^>/{if(seq)print length(seq),name; name=$1; seq=""} !/^>/{seq=seq$0} END{print length(seq),name}' seqs.fa | sort -rn | head -1

# 5. Extract genes on chromosome 1 with score > 800
awk '$1=="chr1" && $5>800' genes.bed

# 6. Count reads per chromosome from a BED file
cut -f1 genes.bed | sort | uniq -c | sort -rn

# 7. Get GC content of sequences (simplified)
grep -v "^>" sequences.fasta | tr -d '\n' | fold -w1 | sort | uniq -c

# 8. Convert multi-line FASTA to single-line
awk '/^>/{if(seq)print seq; print; seq=""} !/^>/{seq=seq$0} END{print seq}' seqs.fa
