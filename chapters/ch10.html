Chapter 10: Sequence Alignment

~90 min Intermediate BLAST, BWA, Bowtie2, SAM/BAM, CIGAR, samtools

Types of Alignment

  • Database search (BLAST) — "What is this sequence similar to?" Query vs. a large database
  • Short-read mapping (BWA, Bowtie2) — Map millions of 75-300 bp reads to a reference genome
  • Long-read mapping (minimap2) — Map PacBio/Nanopore reads (10 kb–1 Mb)
  • Splice-aware alignment (STAR, HISAT2) — For RNA-Seq reads that span exon-exon junctions

BLAST — Basic Local Alignment Search Tool

BLAST finds regions of local similarity between a query sequence and a database. It's the most-used tool in bioinformatics.

# BLAST variants:
# blastn  — nucleotide vs nucleotide
# blastp  — protein vs protein
# blastx  — translated nucleotide vs protein
# tblastn — protein vs translated nucleotide
# tblastx — translated nucleotide vs translated nucleotide

# Step 1: Create a BLAST database
makeblastdb -in reference.fa -dbtype nucl -out ref_db

# Step 2: Run BLAST
blastn -query query.fa -db ref_db \
       -outfmt 6 \
       -evalue 1e-5 \
       -num_threads 4 \
       -out results.txt

# Output format 6 (tabular) columns:
# 1.qseqid  2.sseqid  3.pident  4.length  5.mismatch  6.gapopen
# 7.qstart  8.qend    9.sstart  10.send   11.evalue   12.bitscore

Understanding E-values

The E-value is the number of hits you'd expect by chance in a database of that size:

  • E < 1e-50 — Almost certainly homologous
  • E < 1e-10 — Very likely homologous
  • E < 0.01 — Probably significant
  • E > 1 — Likely a random match

Short-Read Alignment with BWA

BWA-MEM is the standard aligner for Illumina reads against a reference genome.

# Step 1: Index the reference (once per genome)
bwa index reference.fa
# Creates: reference.fa.amb, .ann, .bwt, .pac, .sa

# Step 2: Align paired-end reads
bwa mem -t 8 -R '@RG\tID:sample1\tSM:sample1\tPL:ILLUMINA' \
    reference.fa reads_R1.fastq.gz reads_R2.fastq.gz \
    | samtools sort -@ 4 -o sorted.bam

# Step 3: Index the BAM
samtools index sorted.bam
# Creates: sorted.bam.bai
Read Groups (-R flag)

Always add read group information with -R. GATK and other downstream tools require it. The format is: @RG\tID:id\tSM:sample\tPL:platform

SAM/BAM Format — The Alignment Standard

SAM (Sequence Alignment/Map) is a tab-delimited text format. BAM is its compressed binary equivalent. Every alignment tool outputs SAM/BAM.

SAM Header

@HD  VN:1.6  SO:coordinate                    # File metadata
@SQ  SN:chr1  LN:248956422                     # Reference sequences
@SQ  SN:chr2  LN:242193529
@RG  ID:sample1  SM:sample1  PL:ILLUMINA       # Read groups
@PG  ID:bwa  PN:bwa  VN:0.7.17                 # Programs used

SAM Alignment Fields (11 mandatory columns)

Col  Name   Description
───  ─────  ──────────────────────────────────
1    QNAME  Read name
2    FLAG   Bitwise flag (mapped? paired? strand?)
3    RNAME  Reference chromosome
4    POS    1-based leftmost position
5    MAPQ   Mapping quality (0-60)
6    CIGAR  How the read aligns (matches, insertions, deletions)
7    RNEXT  Mate's chromosome (* if unpaired)
8    PNEXT  Mate's position
9    TLEN   Template length (insert size)
10   SEQ    Read sequence
11   QUAL   Base qualities (same as FASTQ)

SAM Flags — Bitwise Encoding

Flag  Meaning
────  ──────────────────────────────
1     Read is paired
2     Both reads mapped properly
4     Read is unmapped
8     Mate is unmapped
16    Read mapped to reverse strand
32    Mate mapped to reverse strand
64    First in pair (R1)
128   Second in pair (R2)
256   Secondary alignment
1024  PCR/optical duplicate
2048  Supplementary alignment

Common combinations:
0    = single-end, mapped, forward strand
4    = unmapped
16   = mapped, reverse strand
99   = paired, both mapped, mate reverse, first in pair
147  = paired, both mapped, read reverse, second in pair
1024 = PCR duplicate (marked by MarkDuplicates)

CIGAR Strings — Alignment Description

Operation  Meaning
─────────  ──────────────────────────────
M          Match or mismatch (alignment column)
I          Insertion to the reference (bases in read, not in ref)
D          Deletion from the reference (bases in ref, not in read)
N          Skipped region (intron in RNA-Seq spliced alignment)
S          Soft clip (bases in read, not aligned, kept in SEQ)
H          Hard clip (bases not in SEQ at all)

Examples:
50M              50 bases aligned (matches + mismatches)
30M2I18M         30 aligned, 2-base insertion, 18 aligned
25M1D25M         25 aligned, 1-base deletion, 25 aligned
5S45M            5 bases soft-clipped at start, 45 aligned
20M5000N30M      RNA-Seq: 20 aligned, 5kb intron, 30 aligned

Mapping Quality (MAPQ)

MAPQ = -10 × log₁₀(P_wrong_position)

MAPQ  Meaning
────  ──────────────────────────
0     Maps equally well to multiple locations
1-3   Very low confidence
10    90% chance of correct position
20    99% chance
30    99.9% chance
60    Highest confidence (BWA maximum)

# Common filter: keep only high-confidence alignments
samtools view -q 30 sorted.bam

samtools — Your BAM Swiss Army Knife

# Convert SAM to BAM
samtools view -bS aligned.sam -o aligned.bam

# Sort by coordinate (required for most tools)
samtools sort aligned.bam -o sorted.bam

# Index (required for random access)
samtools index sorted.bam

# Alignment statistics
samtools flagstat sorted.bam
# Output: total reads, mapped, paired, duplicates, etc.

# Per-chromosome read counts
samtools idxstats sorted.bam
# Output: chr  length  mapped_reads  unmapped_reads

# Count specific read types
samtools view -c sorted.bam              # Total reads
samtools view -c -F 4 sorted.bam        # Mapped reads (-F excludes flag)
samtools view -c -f 4 sorted.bam        # Unmapped reads (-f requires flag)
samtools view -c -F 1024 sorted.bam     # Non-duplicate reads
samtools view -c -f 1 -F 12 sorted.bam  # Properly paired reads

# Filter by quality and region
samtools view -b -q 30 sorted.bam -o filtered.bam       # MAPQ >= 30
samtools view -b sorted.bam chr1:1000000-2000000 -o region.bam  # Extract region

# Per-base coverage depth
samtools depth sorted.bam | head
# Output: chr  position  depth

# Average coverage
samtools depth sorted.bam | awk '{sum+=$3; n++} END {print "Avg depth:", sum/n}'

# View alignments in a region (text-based viewer)
samtools tview sorted.bam reference.fa
-f vs -F in samtools view

-f 4 = require flag 4 (keep unmapped). -F 4 = exclude flag 4 (keep mapped). Mixing these up is a very common mistake that silently gives wrong results.
