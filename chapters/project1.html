Project 1: Who Is This Microbe?

~2 hours Beginner Unix, FASTQ, QC, BLAST, Species ID

The Biological Question

A clinical microbiology lab receives a patient sample with a suspected bacterial infection. They sequence the isolate on an Illumina MiSeq, producing a FASTQ file. Nobody knows what species it is. Your job: figure it out from the raw reads.

Why does this matter?

Species identification from sequencing data is bread and butter in clinical microbiology. Hospitals need it to pick the right antibiotic. Public health labs need it to trace outbreaks. Food safety labs need it to catch contamination. If bioinformatics has a "Hello World," this is it.

The Dataset

We're using a real whole-genome sequencing run from NCBI's Sequence Read Archive:

Accession:   SRR24043640
Organism:    Unknown (you'll figure this out!)
Platform:    Illumina MiSeq
Layout:      Paired-end, 2 × 250 bp
Size:        ~200 MB compressed
Study:       Clinical isolate sequencing for pathogen identification
Source:      NCBI SRA
Requirements

This project requires a Linux environment with conda installed. You'll need ~2 GB of free disk space and an internet connection to download data. If you haven't set up your environment yet, complete Chapter 2 first.

Step 0: Set Up Your Environment

# Create a project directory
mkdir -p mystery_microbe/{data,results/{qc,trimmed,blast},scripts}
cd mystery_microbe

# Create and activate conda environment
conda create -n microbe-id -y python=3.11
conda activate microbe-id

# Install tools
conda install -c bioconda -c conda-forge -y \
    sra-tools fastqc fastp blast seqkit

# Verify installations
fastqc --version
fastp --version
blastn -version

Step 1: Download the Data

First, grab the raw sequencing reads from NCBI. Think of SRA as a giant library of sequencing experiments anyone can access.

# Download from SRA
cd data
fasterq-dump SRR24043640 --split-files --threads 4

# Check what we got
ls -lh
# Expected: SRR24043640_1.fastq  SRR24043640_2.fastq

# Quick look at the first read
head -4 SRR24043640_1.fastq

You should see 4 lines: a header starting with @, a DNA sequence, a + separator, and quality scores. That's one read. The entire file is millions of these stacked together.

# How many reads do we have?
echo "Forward reads: $(( $(wc -l < SRR24043640_1.fastq) / 4 ))"
echo "Reverse reads: $(( $(wc -l < SRR24043640_2.fastq) / 4 ))"
Expected output

You should see roughly 500K to 1M read pairs. That's typical for a bacterial WGS run on MiSeq.

Step 2: Quality Control

Before doing anything with the data, check if it's any good. FastQC generates a visual report showing quality scores, adapter contamination, GC bias, and other red flags. This takes about a minute.

# Run FastQC on both files
cd ../results/qc
fastqc ../../data/SRR24043640_1.fastq ../../data/SRR24043640_2.fastq -o . -t 2

# List the reports
ls *.html

What to look for in the FastQC report:

  • Per base sequence quality should be mostly green (>Q20). Quality usually drops toward the 3' end
  • Adapter content should be low. If it's high, trimming takes care of it
  • GC content is a clue about the organism. E. coli sits around 50%, Mycobacterium around 65%, Staphylococcus around 33%
  • Sequence length distribution should be around 250 bp for MiSeq 2x250

Step 3: Trim and Filter

Sequencing isn't perfect. Reads often have low-quality bases at the ends and leftover adapter sequences. fastp cleans all of this up in one pass. After this step, you'll have smaller but much higher quality files.

cd ../trimmed
fastp \
    -i ../../data/SRR24043640_1.fastq \
    -I ../../data/SRR24043640_2.fastq \
    -o trimmed_R1.fastq \
    -O trimmed_R2.fastq \
    --qualified_quality_phred 20 \
    --length_required 100 \
    --detect_adapter_for_pe \
    --thread 4 \
    --html fastp_report.html \
    --json fastp_report.json

# Check how many reads survived
echo "Before: $(( $(wc -l < ../../data/SRR24043640_1.fastq) / 4 )) read pairs"
echo "After:  $(( $(wc -l < trimmed_R1.fastq) / 4 )) read pairs"
What fastp does
  • --qualified_quality_phred 20 trims bases below Q20 (99% accuracy)
  • --length_required 100 drops reads shorter than 100 bp after trimming
  • --detect_adapter_for_pe auto-detects and removes Illumina adapters

Open fastp_report.html in a browser to see a visual summary of what was trimmed.

Step 4: Subsample for BLAST

Here's a practical trick: you don't need to BLAST a million reads to identify a species. A random sample of 1000 is more than enough, and it runs in seconds instead of hours.

# Take 1000 random reads
cd ../blast
seqkit sample -n 1000 -s 42 ../trimmed/trimmed_R1.fastq > subset_1000.fastq

# Convert to FASTA (BLAST needs FASTA, not FASTQ)
seqkit fq2fa subset_1000.fastq > subset_1000.fasta

# Verify
grep -c "^>" subset_1000.fasta
# Expected: 1000

Step 5: BLAST Against NCBI

This is the moment of truth. You're taking your mystery sequences and asking NCBI: "What organism do these come from?" BLAST compares each read against billions of known sequences and returns the closest matches. There are three ways to do this:

Option A: Web BLAST (no local database needed)

# Take just 10 reads for web BLAST (it's slow for many sequences)
head -20 subset_1000.fasta > query_10.fasta

# Go to https://blast.ncbi.nlm.nih.gov/Blast.cgi
# Select: blastn → Nucleotide collection (nt)
# Paste or upload query_10.fasta
# Click BLAST and wait ~2 minutes

Option B: Command-line BLAST (faster, needs local database)

# If you have a local nt database:
blastn \
    -query subset_1000.fasta \
    -db nt \
    -outfmt "6 qseqid sseqid pident length evalue stitle" \
    -evalue 1e-10 \
    -max_target_seqs 1 \
    -num_threads 4 \
    -out blast_results.txt

# Count hits per species
awk -F'\t' '{print $6}' blast_results.txt | \
    sed 's/ .*//' | \
    sort | uniq -c | sort -rn | head -10

Option C: Use Biopython for remote BLAST

from Bio.Blast import NCBIWWW, NCBIXML
from Bio import SeqIO

# Read first 5 sequences
records = list(SeqIO.parse("subset_1000.fasta", "fasta"))[:5]

for record in records:
    print(f"BLASTing {record.id}...")
    result = NCBIWWW.qblast("blastn", "nt", str(record.seq))
    blast_records = NCBIXML.parse(result)
    blast_record = next(blast_records)
    
    if blast_record.alignments:
        top_hit = blast_record.alignments[0]
        print(f"  Top hit: {top_hit.title[:100]}")
        print(f"  E-value: {top_hit.hsps[0].expect}")
        print(f"  Identity: {top_hit.hsps[0].identities}/{top_hit.hsps[0].align_length}")
    print()

Step 6: Interpret Your Results

This is where the biology happens. Look at your BLAST results. You should see a clear pattern: most reads hitting the same organism with >95% identity and E-values well below 1e-50. If the top hits are scattered across different species, something might be off with your data (contamination, mixed culture, etc.).

How to interpret BLAST results
  • % Identity > 97% means same species
  • % Identity 90-97% means same genus, different species
  • % Identity < 90% means different genus
  • E-value < 1e-50 means a very confident match
  • Multiple reads hitting the same organism is strong evidence
