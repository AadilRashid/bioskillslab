Project 1: Who Is This Microbe?

~2 hours Beginner Unix, FASTQ, QC, BLAST, Species ID

The Biological Question

A clinical microbiology lab receives a patient sample with a suspected bacterial infection. They sequence the isolate using Illumina, producing a FASTQ file — but they don't know what species it is. Your job: identify the organism from raw sequencing data.

Why does this matter?

Species identification from sequencing data is one of the most fundamental tasks in clinical microbiology, outbreak investigation, food safety, and environmental monitoring. Hospitals use this to determine which antibiotic to prescribe. Public health agencies use it to trace outbreaks. It's the "Hello World" of bioinformatics — simple, practical, and immediately useful.

The Dataset

We'll use a real whole-genome sequencing run from NCBI's Sequence Read Archive:

Accession:   SRR24043640
Organism:    Unknown (you'll figure this out!)
Platform:    Illumina MiSeq
Layout:      Paired-end, 2 × 250 bp
Size:        ~200 MB compressed
Study:       Clinical isolate sequencing for pathogen identification
Source:      NCBI SRA
Requirements

This project requires a Linux environment with conda installed. You'll need ~2 GB of free disk space and an internet connection to download data. If you haven't set up your environment yet, complete Chapter 2 first.

Step 0: Set Up Your Environment

# Create a project directory
mkdir -p mystery_microbe/{data,results/{qc,trimmed,blast},scripts}
cd mystery_microbe

# Create and activate conda environment
conda create -n microbe-id -y python=3.11
conda activate microbe-id

# Install tools
conda install -c bioconda -c conda-forge -y \
    sra-tools fastqc fastp blast seqkit

# Verify installations
fastqc --version
fastp --version
blastn -version

Step 1: Download the Data

# Download from SRA
cd data
fasterq-dump SRR24043640 --split-files --threads 4

# Check what we got
ls -lh
# Expected: SRR24043640_1.fastq  SRR24043640_2.fastq

# Quick look at the first read
head -4 SRR24043640_1.fastq

You should see 4 lines: a header starting with @, a DNA sequence, a + separator, and quality scores.

# How many reads do we have?
echo "Forward reads: $(( $(wc -l < SRR24043640_1.fastq) / 4 ))"
echo "Reverse reads: $(( $(wc -l < SRR24043640_2.fastq) / 4 ))"
Expected output

You should see roughly 500,000–1,000,000 read pairs. This is typical for a bacterial whole-genome sequencing run on MiSeq.

Step 2: Quality Control

Never skip QC. We need to know if the data is good before we analyze it.

# Run FastQC on both files
cd ../results/qc
fastqc ../../data/SRR24043640_1.fastq ../../data/SRR24043640_2.fastq -o . -t 2

# List the reports
ls *.html

What to check in the FastQC report:

  • Per base sequence quality — Should be mostly green (>Q20). Quality typically drops at the 3' end of reads
  • Adapter content — Should be low. If high, trimming will remove them
  • GC content — The peak tells you something about the organism. E. coli is ~50%, Mycobacterium is ~65%, Staphylococcus is ~33%
  • Sequence length distribution — Should be ~250 bp for MiSeq 2×250

Step 3: Trim and Filter

Remove adapters, low-quality bases, and short reads:

cd ../trimmed
fastp \
    -i ../../data/SRR24043640_1.fastq \
    -I ../../data/SRR24043640_2.fastq \
    -o trimmed_R1.fastq \
    -O trimmed_R2.fastq \
    --qualified_quality_phred 20 \
    --length_required 100 \
    --detect_adapter_for_pe \
    --thread 4 \
    --html fastp_report.html \
    --json fastp_report.json

# Check how many reads survived
echo "Before: $(( $(wc -l < ../../data/SRR24043640_1.fastq) / 4 )) read pairs"
echo "After:  $(( $(wc -l < trimmed_R1.fastq) / 4 )) read pairs"
What fastp does
  • --qualified_quality_phred 20 — Bases below Q20 (99% accuracy) are trimmed
  • --length_required 100 — Reads shorter than 100 bp after trimming are discarded
  • --detect_adapter_for_pe — Automatically detects and removes Illumina adapters

Open fastp_report.html in a browser to see a visual summary of what was trimmed.

Step 4: Subsample for BLAST

BLAST against the full NCBI nt database is slow. For species identification, we don't need all reads — a random subset is enough:

# Take 1000 random reads
cd ../blast
seqkit sample -n 1000 -s 42 ../trimmed/trimmed_R1.fastq > subset_1000.fastq

# Convert to FASTA (BLAST needs FASTA, not FASTQ)
seqkit fq2fa subset_1000.fastq > subset_1000.fasta

# Verify
grep -c "^>" subset_1000.fasta
# Expected: 1000

Step 5: BLAST Against NCBI

Now the key step — search these sequences against NCBI's nucleotide database to identify the organism:

Option A: Web BLAST (no local database needed)

# Take just 10 reads for web BLAST (it's slow for many sequences)
head -20 subset_1000.fasta > query_10.fasta

# Go to https://blast.ncbi.nlm.nih.gov/Blast.cgi
# Select: blastn → Nucleotide collection (nt)
# Paste or upload query_10.fasta
# Click BLAST and wait ~2 minutes

Option B: Command-line BLAST (faster, needs local database)

# If you have a local nt database:
blastn \
    -query subset_1000.fasta \
    -db nt \
    -outfmt "6 qseqid sseqid pident length evalue stitle" \
    -evalue 1e-10 \
    -max_target_seqs 1 \
    -num_threads 4 \
    -out blast_results.txt

# Count hits per species
awk -F'\t' '{print $6}' blast_results.txt | \
    sed 's/ .*//' | \
    sort | uniq -c | sort -rn | head -10

Option C: Use Biopython for remote BLAST

from Bio.Blast import NCBIWWW, NCBIXML
from Bio import SeqIO

# Read first 5 sequences
records = list(SeqIO.parse("subset_1000.fasta", "fasta"))[:5]

for record in records:
    print(f"BLASTing {record.id}...")
    result = NCBIWWW.qblast("blastn", "nt", str(record.seq))
    blast_records = NCBIXML.parse(result)
    blast_record = next(blast_records)
    
    if blast_record.alignments:
        top_hit = blast_record.alignments[0]
        print(f"  Top hit: {top_hit.title[:100]}")
        print(f"  E-value: {top_hit.hsps[0].expect}")
        print(f"  Identity: {top_hit.hsps[0].identities}/{top_hit.hsps[0].align_length}")
    print()

Step 6: Interpret Your Results

Look at your BLAST results. You should see a clear pattern — most reads hitting the same organism with >95% identity and very low E-values (< 1e-50).

How to interpret BLAST results
  • % Identity > 97% — Same species
  • % Identity 90-97% — Same genus, different species
  • % Identity < 90% — Different genus
  • E-value < 1e-50 — Very confident match
  • Multiple reads hitting the same organism — Strong evidence
