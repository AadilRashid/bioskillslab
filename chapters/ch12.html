Chapter 12: RNA-Seq Analysis

~90 min Advanced STAR, featureCounts, DESeq2, edgeR, Pathway Analysis

What is RNA-Seq?

RNA-Seq measures gene expression by sequencing the RNA (transcriptome) in a sample. The core question: which genes are turned on/off, and by how much, between conditions?

Applications: differential expression, alternative splicing, novel transcript discovery, allele-specific expression, single-cell transcriptomics.

Experimental Design — Get This Right First

  • Biological replicates — Minimum 3 per condition, ideally 5+. More replicates = more statistical power
  • Sequencing depth — 20-30 million reads per sample for most DE analyses. More for rare transcripts or splicing
  • Read length — 75-150 bp paired-end is standard. Longer reads help with isoform detection
  • Batch effects — Process all samples together. If batching is unavoidable, balance conditions across batches
  • Controls — Include appropriate biological controls. Consider spike-ins (ERCC) for normalization validation
The Most Common RNA-Seq Mistake

Running 1-2 replicates per condition. With only 2 replicates, you have almost no statistical power to detect DE genes. The money spent on deeper sequencing of 2 samples is better spent on shallower sequencing of 4+ samples.

Step 3: Alignment with STAR

STAR is the most widely used RNA-Seq aligner. It's splice-aware — it can align reads that span exon-exon junctions across introns.

# Generate genome index (once per genome — needs ~30 GB RAM for human)
STAR --runMode genomeGenerate \
     --genomeDir star_index/ \
     --genomeFastaFiles Homo_sapiens.GRCh38.dna.primary_assembly.fa \
     --sjdbGTFfile Homo_sapiens.GRCh38.110.gtf \
     --sjdbOverhang 149 \
     --runThreadN 8

# Align reads (per sample)
STAR --genomeDir star_index/ \
     --readFilesIn sample_R1.fastq.gz sample_R2.fastq.gz \
     --readFilesCommand zcat \
     --outSAMtype BAM SortedByCoordinate \
     --quantMode GeneCounts \
     --outFileNamePrefix results/star/sample_ \
     --runThreadN 8

# Index the output BAM
samtools index results/star/sample_Aligned.sortedByCoord.out.bam

Step 4: Counting with featureCounts

# Count reads per gene for all samples at once
featureCounts \
    -a Homo_sapiens.GRCh38.110.gtf \
    -o results/counts/gene_counts.txt \
    -T 4 \
    -p --countReadPairs \
    -s 2 \
    results/star/*_Aligned.sortedByCoord.out.bam

# -a: annotation GTF
# -p --countReadPairs: count fragments, not individual reads (paired-end)
# -s 2: strand-specific (2 = reverse, most common for Illumina dUTP protocol)
# -T: threads

Step 5: Differential Expression with DESeq2

library(DESeq2)
library(dplyr)

# Load count matrix (skip featureCounts metadata columns)
raw <- read.delim("results/counts/gene_counts.txt", comment.char="#")
counts <- raw[, 7:ncol(raw)]  # Columns 7+ are the sample counts
rownames(counts) <- raw$Geneid
colnames(counts) <- gsub(".*star\\.|\\.Aligned.*", "", colnames(counts))

# Sample metadata
coldata <- data.frame(
  condition = factor(c("control","control","control","treatment","treatment","treatment")),
  row.names = colnames(counts)
)

# Create DESeq2 object
dds <- DESeqDataSetFromMatrix(countData=counts, colData=coldata, design=~condition)

# Pre-filter: remove genes with very low counts
keep <- rowSums(counts(dds)) >= 10
dds <- dds[keep, ]
cat("Genes after filtering:", nrow(dds), "\n")

# Run DESeq2
dds <- DESeq(dds)

# Extract results
res <- results(dds, contrast=c("condition","treatment","control"), alpha=0.05)
res <- res[order(res$padj), ]

# Summary
summary(res)
sig <- subset(res, padj < 0.05 & abs(log2FoldChange) > 1)
cat(nrow(sig), "significant DE genes\n")
cat(sum(sig$log2FoldChange > 0), "upregulated,", sum(sig$log2FoldChange < 0), "downregulated\n")

Step 6: Visualization

library(ggplot2)

# Volcano plot
df <- as.data.frame(res) %>% mutate(sig = !is.na(padj) & padj < 0.05 & abs(log2FoldChange) > 1)
ggplot(df, aes(log2FoldChange, -log10(padj), color=sig)) +
  geom_point(size=0.5, alpha=0.5) +
  scale_color_manual(values=c("grey70","red3")) +
  geom_hline(yintercept=-log10(0.05), linetype="dashed") +
  geom_vline(xintercept=c(-1,1), linetype="dashed") +
  theme_minimal() + labs(title="Volcano Plot")

# PCA — do samples cluster by condition?
vsd <- vst(dds)
plotPCA(vsd, intgroup="condition") + theme_minimal()

# Heatmap of top 50 DE genes
library(pheatmap)
top50 <- head(rownames(res[order(res$padj),]), 50)
mat <- assay(vsd)[top50, ]
pheatmap(mat, scale="row", annotation_col=coldata,
         color=colorRampPalette(c("navy","white","firebrick3"))(100))

Alternative: Pseudo-alignment with Salmon

Salmon skips traditional alignment entirely — it quantifies transcript abundance directly from FASTQ files. It's 10-100x faster than STAR + featureCounts.

# Build index (once)
salmon index -t transcripts.fa -i salmon_index

# Quantify (per sample)
salmon quant -i salmon_index \
    -l A \
    -1 sample_R1.fastq.gz -2 sample_R2.fastq.gz \
    -o results/salmon/sample/ \
    --threads 8

# Import into R with tximeta (for DESeq2)
library(tximeta)
coldata$files <- file.path("results/salmon", coldata$sample, "quant.sf")
se <- tximeta(coldata)
dds <- DESeqDataSet(se, design=~condition)
