Chapter 2: Setting Up Your Environment

~50 min Beginner Linux, WSL, Conda, Project Structure, Markdown

Why Linux?

Almost all bioinformatics tools are built for Unix/Linux. The reasons are historical and practical:

  • Servers and HPC clusters run Linux (you'll SSH into them)
  • Cloud instances (AWS, GCP) default to Linux
  • Most tools (samtools, BWA, GATK) are command-line only
  • Unix pipes and text processing are ideal for bioinformatics data
  • Package managers (Conda, apt) make installing tools easy

Getting a Linux Environment

Windows users → WSL2 (Windows Subsystem for Linux). This gives you a full Ubuntu installation inside Windows with near-native performance.

# Open PowerShell as Administrator:
wsl --install

# This installs Ubuntu by default. Restart your computer.
# Open "Ubuntu" from Start menu → set username and password.

# Verify it works:
lsb_release -a     # Should show Ubuntu
uname -r            # Should show a Linux kernel version

macOS users — Your terminal is already Unix-based. Install Homebrew for package management:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install wget samtools

Native Linux — You're already set. Ubuntu or CentOS are most common in bioinformatics.

Package Management with Conda

Conda solves the biggest headache in bioinformatics: dependency hell. Tool A needs Python 3.8, Tool B needs Python 3.11, Tool C needs a specific version of libhts. Conda manages all of this with isolated environments.

Installing Miniconda

# Download Miniconda (lightweight — recommended over full Anaconda)
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh
# Follow prompts, say "yes" to init. Restart terminal.

# Verify
conda --version

Setting Up Bioconda

Bioconda is a Conda channel with 8000+ bioinformatics packages. Set it up once:

conda config --add channels defaults
conda config --add channels bioconda
conda config --add channels conda-forge
conda config --set channel_priority strict

Creating and Managing Environments

# Create a new environment
conda create -n rnaseq python=3.11
conda activate rnaseq

# Install tools into the active environment
conda install -c bioconda samtools=1.17 bwa fastqc star subread

# See what's installed
conda list

# Deactivate (go back to base)
conda deactivate

# List all environments
conda env list

# Remove an environment
conda env remove -n old_project

Reproducibility: Exporting Environments

# Export exact environment (for reproducing on same OS)
conda env export > environment.yml

# Export cross-platform (just package names)
conda env export --from-history > environment_portable.yml

# Recreate from file
conda env create -f environment.yml
Mamba — Faster Conda

Conda can be slow at resolving dependencies. Mamba is a drop-in replacement that's 10-100x faster:

conda install -n base -c conda-forge mamba
mamba create -n myenv samtools bwa    # Same syntax, much faster

Project Directory Structure

A well-organized project is a reproducible project. Before you write a single line of code, set up your directory structure:

my_rnaseq_project/
├── README.md              # What this project is, how to run it
├── data/
│   ├── raw/               # Original FASTQ files (NEVER modify these)
│   ├── processed/         # Trimmed reads, filtered data
│   └── reference/         # Reference genome, GTF annotation
├── results/
│   ├── qc/                # FastQC reports
│   ├── alignments/        # BAM files
│   ├── counts/            # Gene count matrices
│   ├── de/                # Differential expression results
│   ├── figures/           # Publication-ready plots
│   └── tables/            # Summary tables
├── scripts/
│   ├── 01_qc.sh           # Quality control
│   ├── 02_trim.sh         # Adapter trimming
│   ├── 03_align.sh        # Read alignment
│   ├── 04_count.sh        # Gene counting
│   └── 05_de_analysis.R   # Differential expression
├── envs/
│   └── environment.yml    # Conda environment specification
├── docs/
│   └── analysis_notes.md  # Lab notebook / analysis decisions
├── Snakefile              # Workflow definition (optional but recommended)
└── .gitignore             # Files Git should ignore

Create It in One Command

mkdir -p my_project/{data/{raw,processed,reference},results/{qc,alignments,counts,de,figures,tables},scripts,envs,docs}
The data/raw/ Rule

Files in data/raw/ are sacred. Never modify them. Never write to that directory. If you need to change something, write the output to data/processed/. This ensures you can always re-run your entire analysis from scratch.

Numbering Your Scripts

Prefix scripts with numbers (01_, 02_, etc.) to indicate execution order. Anyone looking at your project immediately knows the workflow:

scripts/
├── 01_download_data.sh     # Step 1
├── 02_qc.sh                # Step 2
├── 03_trim.sh              # Step 3
├── 04_align.sh             # Step 4
├── 05_count.sh             # Step 5
└── 06_de_analysis.R        # Step 6

Text Editors

  • nano — Simple, beginner-friendly. nano file.txt → edit → Ctrl+O save → Ctrl+X exit
  • vim — Powerful, steep learning curve. Press i to type, Esc then :wq to save and quit
  • VS Code + Remote SSH — Best of both worlds. GUI editor on your laptop, files on the server

Documenting with Markdown

Every project needs a README.md. Markdown is simple, readable as plain text, and renders beautifully on GitHub:

# RNA-Seq Analysis of Drug Treatment

## Overview
Differential expression analysis comparing treated vs control samples
in HeLa cells. 3 biological replicates per condition.

## Data
- Raw reads: `data/raw/` (downloaded from SRA, accession PRJNA123456)
- Reference: GRCh38 from Ensembl release 110

## Methods
1. QC: FastQC v0.12.1
2. Trimming: fastp v0.23.4 (min quality 20, min length 50)
3. Alignment: STAR v2.7.11a (2-pass mode)
4. Counting: featureCounts (Subread v2.0.6)
5. DE: DESeq2 v1.40.2 (padj < 0.05, |log2FC| > 1)

## How to Reproduce
```bash
conda env create -f envs/environment.yml
conda activate rnaseq
snakemake --cores 8
```

## Results
- 342 upregulated, 287 downregulated genes
- See `results/figures/volcano_plot.pdf`
