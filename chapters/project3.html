Project 3: Find Mutations in a Cancer Sample

~4 hours Advanced BWA, GATK, VCF, SnpEff, Clinical Genomics

The Biological Question

A patient is diagnosed with chronic lymphocytic leukemia (CLL). Their tumor DNA is sequenced alongside a matched normal sample. Your job: find the somatic mutations that may be driving the cancer and identify any clinically actionable variants.

Why does this matter?

Variant calling is the foundation of precision medicine. Identifying mutations in a patient's tumor determines which targeted therapies might work, predicts prognosis, and guides clinical decisions. Every cancer genomics study, rare disease diagnosis, and pharmacogenomics test relies on this pipeline.

The Dataset

We'll use data from the GATK Best Practices tutorial — a well-characterized sample with known variants for validation:

Source:      GATK Resource Bundle
Sample:      NA12878 (HapMap/1000 Genomes reference sample)
Type:        Whole exome sequencing (WES)
Platform:    Illumina HiSeq
Reference:   GRCh38 / hg38
Truth set:   GIAB high-confidence variant calls (for validation)
Why NA12878?

NA12878 is the most well-characterized human genome in existence. The Genome in a Bottle (GIAB) consortium has produced high-confidence variant calls for this sample, making it the gold standard for benchmarking variant calling pipelines. If your pipeline produces results that match GIAB, you know it's working correctly.

Step 3: Alignment (if starting from FASTQ)

REF="../reference/Homo_sapiens_assembly38.fasta"

# Align with BWA-MEM
bwa mem -t 8 \
    -R '@RG\tID:NA12878\tSM:NA12878\tPL:ILLUMINA\tLB:lib1' \

samtools index ../results/aligned/NA12878.sorted.bam

# Alignment stats
samtools flagstat ../results/aligned/NA12878.sorted.bam

Step 4: Mark Duplicates

cd ../../results/aligned

gatk MarkDuplicates \
    -I NA12878.sorted.bam \
    -O NA12878.dedup.bam \
    -M dedup_metrics.txt \
    --REMOVE_DUPLICATES false

samtools index NA12878.dedup.bam

# Check duplication rate
grep "PERCENT_DUPLICATION" dedup_metrics.txt
# Typical WES: 10-30% duplicates

Step 5: Base Quality Score Recalibration (BQSR)

REF="../../data/reference/Homo_sapiens_assembly38.fasta"
DBSNP="../../data/known_sites/Homo_sapiens_assembly38.dbsnp138.vcf"
INDELS="../../data/known_sites/Homo_sapiens_assembly38.known_indels.vcf.gz"

# Build recalibration model
gatk BaseRecalibrator \
    -I NA12878.dedup.bam \
    -R ${REF} \
    --known-sites ${DBSNP} \
    --known-sites ${INDELS} \
    -O recal_table

# Apply recalibration
gatk ApplyBQSR \
    -I NA12878.dedup.bam \
    -R ${REF} \
    --bqsr-recal-file recal_table \
    -O NA12878.recal.bam

samtools index NA12878.recal.bam

Step 6: Call Variants

# Call variants with HaplotypeCaller
gatk HaplotypeCaller \
    -I NA12878.recal.bam \
    -R ${REF} \
    -L chr20 \
    -O ../variants/NA12878.raw.vcf.gz

# How many raw variants?
bcftools stats ../variants/NA12878.raw.vcf.gz | grep "^SN"

Step 7: Filter Variants

cd ../variants

# Separate SNPs and indels
gatk SelectVariants -V NA12878.raw.vcf.gz --select-type-to-include SNP -O raw_snps.vcf.gz
gatk SelectVariants -V NA12878.raw.vcf.gz --select-type-to-include INDEL -O raw_indels.vcf.gz

# Hard filter SNPs
gatk VariantFiltration -V raw_snps.vcf.gz \
    --filter-expression "QD < 2.0" --filter-name "LowQD" \
    --filter-expression "MQ < 40.0" --filter-name "LowMQ" \
    --filter-expression "FS > 60.0" --filter-name "HighFS" \
    --filter-expression "SOR > 3.0" --filter-name "HighSOR" \
    -O filtered_snps.vcf.gz

# Hard filter indels
gatk VariantFiltration -V raw_indels.vcf.gz \
    --filter-expression "QD < 2.0" --filter-name "LowQD" \
    --filter-expression "FS > 200.0" --filter-name "HighFS" \
    -O filtered_indels.vcf.gz

# Keep only PASS variants
bcftools view -f PASS filtered_snps.vcf.gz -o pass_snps.vcf.gz
bcftools view -f PASS filtered_indels.vcf.gz -o pass_indels.vcf.gz

# Merge
bcftools concat pass_snps.vcf.gz pass_indels.vcf.gz | bcftools sort -o final_variants.vcf.gz
bcftools index final_variants.vcf.gz

# Summary
echo "=== Final Variant Summary ==="
echo "SNPs:   $(bcftools view -v snps final_variants.vcf.gz | grep -vc '^#')"
echo "Indels: $(bcftools view -v indels final_variants.vcf.gz | grep -vc '^#')"
echo "Ti/Tv:  $(bcftools stats final_variants.vcf.gz | grep 'Ts/Tv' | head -1)"
Quality check: Ti/Tv ratio

For whole exome data, the transition/transversion (Ti/Tv) ratio should be ~2.8-3.2. For whole genome, ~2.0-2.1. If your ratio is much lower, your filtering may be too lenient. If much higher, too strict.

Step 8: Annotate Variants

cd ../annotated

# Annotate with SnpEff
snpEff ann GRCh38.105 ../variants/final_variants.vcf.gz > annotated.vcf

# Summary of variant effects
grep -v "^#" annotated.vcf | \
    awk -F'|' '{print $2}' | \
    sort | uniq -c | sort -rn | head -15

# Extract high-impact variants (frameshift, stop gained, splice)
SnpSift filter "ANN[0].IMPACT = 'HIGH'" annotated.vcf > high_impact.vcf
echo "High-impact variants: $(grep -vc '^#' high_impact.vcf)"

# Extract missense variants
SnpSift filter "ANN[0].EFFECT has 'missense'" annotated.vcf > missense.vcf
echo "Missense variants: $(grep -vc '^#' missense.vcf)"

Step 9: Validate Against Truth Set

# Download GIAB truth set for NA12878 (chr20)
wget https://ftp-trace.ncbi.nlm.nih.gov/giab/ftp/release/NA12878_HG001/latest/GRCh38/HG001_GRCh38_1_22_v4.2.1_benchmark.vcf.gz
wget https://ftp-trace.ncbi.nlm.nih.gov/giab/ftp/release/NA12878_HG001/latest/GRCh38/HG001_GRCh38_1_22_v4.2.1_benchmark.vcf.gz.tbi

# Compare your calls to truth
bcftools isec -p comparison \
    ../variants/final_variants.vcf.gz \
    HG001_GRCh38_1_22_v4.2.1_benchmark.vcf.gz \
    -r chr20

echo "=== Validation ==="
echo "Your calls only:  $(grep -vc '^#' comparison/0000.vcf)"
echo "Truth only:        $(grep -vc '^#' comparison/0001.vcf)"
echo "Both (concordant): $(grep -vc '^#' comparison/0002.vcf)"

# Calculate sensitivity and precision
CONCORDANT=$(grep -vc '^#' comparison/0002.vcf)
YOUR_ONLY=$(grep -vc '^#' comparison/0000.vcf)
TRUTH_ONLY=$(grep -vc '^#' comparison/0001.vcf)
echo "Sensitivity: $(echo "scale=3; $CONCORDANT / ($CONCORDANT + $TRUTH_ONLY)" | bc)"
echo "Precision:   $(echo "scale=3; $CONCORDANT / ($CONCORDANT + $YOUR_ONLY)" | bc)"
Expected performance

A well-tuned GATK pipeline on NA12878 should achieve >99% sensitivity and >99% precision for SNPs. If your numbers are significantly lower, check your filtering parameters or alignment quality.
