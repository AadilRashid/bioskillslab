Chapter 5: P-Values — The Most Misunderstood Number in Science

~45 min Beginner P-value, Null Hypothesis, Type I Error, Significance

What a P-Value Actually Is

A p-value answers one specific question: if there were no real effect, how likely would I be to see data this extreme or more extreme?

That's it. Nothing more. It's the probability of the data given the null hypothesis, not the probability of the hypothesis given the data.

A concrete example

You compare gene expression between 4 treated and 4 control mice. The treated group has higher expression. You run a t-test and get p = 0.03.

This means: if the treatment had zero effect (null hypothesis), there's a 3% chance you'd see a difference this large just from random sampling variation. That's pretty unlikely, so you suspect the treatment actually does something.

What a P-Value is NOT

Most people get this wrong. Even many published papers get it wrong.

Common misinterpretations
  • NOT the probability that the null hypothesis is true
  • NOT the probability that your result is a fluke
  • NOT the probability that you'll get the same result if you repeat the experiment
  • NOT a measure of effect size (p = 0.001 doesn't mean a big effect)

A small p-value means the data is unlikely under the null hypothesis. It does NOT tell you how likely the alternative hypothesis is. Those are completely different questions.

The Null Hypothesis

Every p-value is calculated against a null hypothesis. Usually the null says "there's no difference" or "there's no effect."

Differential expression:  H₀: log2FC = 0 (no difference between groups)
Variant calling:          H₀: this position matches the reference
Correlation:              H₀: r = 0 (no linear relationship)
GWAS:                     H₀: this SNP has no association with the trait

The p-value tells you how surprising your data would be if the null were true. If it's very surprising (small p), you reject the null.

The 0.05 Threshold

The convention is to call results "statistically significant" if p < 0.05. This threshold is completely arbitrary. Ronald Fisher picked it in the 1920s because it felt reasonable. There's nothing magical about it.

p = 0.049 and p = 0.051 are practically identical. Treating one as "significant" and the other as "not significant" is absurd, but it happens in papers every day.

Better practice

Report the actual p-value, not just "significant" or "not significant." Let readers judge for themselves. And always report the effect size (fold change, correlation coefficient, etc.) alongside the p-value. A tiny effect can have a tiny p-value if you have enough data.

Small P-Value ≠ Big Effect

This trips up a lot of people. With enough samples, even a trivially small difference becomes "statistically significant."

Study A: n = 5 per group,  log2FC = 3.0,  p = 0.08   (not significant)
Study B: n = 500 per group, log2FC = 0.1, p = 0.001  (highly significant)

Study A found a large effect but couldn't confirm it (too few samples). Study B found a tiny, biologically meaningless effect but confirmed it with high confidence (lots of samples). Which result is more interesting biologically? Study A, obviously.

P-values mix up effect size and sample size. Always look at both.

Type I and Type II Errors

                    Reality
                    No effect    Real effect
Your decision:
  "Significant"     Type I ✗     Correct ✓
  "Not significant" Correct ✓    Type II ✗
  • Type I error (false positive): You claim an effect exists when it doesn't. Controlled by your significance threshold (α, usually 0.05).
  • Type II error (false negative): You miss a real effect. Related to statistical power (Chapter 14).

In genomics, Type I errors are especially dangerous because you're testing thousands of genes simultaneously. Testing 20,000 genes at α = 0.05 gives you 1,000 expected false positives. This is why multiple testing correction exists (Chapter 7).

P-Values in Bioinformatics

DESeq2 output

# DESeq2 gives you two p-value columns:
# pvalue  — raw p-value from the Wald test
# padj   — adjusted p-value (Benjamini-Hochberg FDR)

# ALWAYS use padj, never pvalue
sig <- subset(res, padj < 0.05 & abs(log2FoldChange) > 1)

BLAST E-values

BLAST reports E-values, not p-values. The E-value is the expected number of hits with this score or better that you'd find by chance in a database of that size. E = 1e-50 means you'd need to search 10⁵⁰ random databases to find one hit this good by chance. Very confident.

GWAS

Genome-wide association studies test millions of SNPs. The significance threshold is p < 5 × 10⁻⁸ (not 0.05) to account for the massive number of tests.

The Bottom Line

  • A p-value is the probability of your data (or more extreme) under the null hypothesis
  • Small p means the data is surprising if there's no effect
  • It does NOT tell you the probability that the effect is real
  • Always report effect size alongside p-values
  • The 0.05 threshold is a convention, not a law of nature
  • In genomics, always use adjusted p-values (FDR) because you're testing thousands of things
