Chapter 7: Multiple Testing — The FDR Problem

~45 min Intermediate FDR, Bonferroni, Benjamini-Hochberg, padj

The Problem

You test one gene at p < 0.05. There's a 5% chance of a false positive. Acceptable.

You test 20,000 genes at p < 0.05. You expect 1,000 false positives. Not acceptable.

This is the multiple testing problem. Every time you run another test, you roll the dice again. Run enough tests and you're guaranteed to find "significant" results even when nothing is going on.

It's everywhere in genomics

RNA-Seq DE:     ~20,000 genes tested
GWAS:           ~1,000,000 SNPs tested
Single-cell:    ~20,000 genes × dozens of clusters
ChIP-Seq:       ~100,000 peaks tested
Metagenomics:   ~10,000 species tested

If you don't correct for multiple testing, your results are mostly noise.

Solution 1: Bonferroni Correction

The simplest fix. Divide your significance threshold by the number of tests.

Original threshold:  α = 0.05
Number of tests:     m = 20,000
Bonferroni threshold: α/m = 0.05/20,000 = 2.5 × 10⁻⁶

Only call a gene significant if p < 0.0000025. This controls the family-wise error rate (FWER): the probability of even one false positive across all tests.

The problem with Bonferroni

It's too conservative. By guarding so aggressively against false positives, you miss a lot of real effects. With 20,000 tests, Bonferroni demands extremely small p-values. Many truly differentially expressed genes won't make the cut.

Solution 2: False Discovery Rate (FDR)

Instead of controlling the chance of ANY false positive, FDR controls the PROPORTION of false positives among your significant results.

Bonferroni (FWER): "I want less than 5% chance of even ONE false positive"
FDR:               "I accept that up to 5% of my significant results might be false"

FDR is much less conservative. If you call 500 genes significant at FDR < 0.05, you expect about 25 of them (5%) to be false positives. The other 475 are real. That's a much better deal than Bonferroni, which might only find 50 genes total.

Benjamini-Hochberg Procedure

The standard method for FDR correction. This is what DESeq2, edgeR, and most genomics tools use.

How it works:
1. Rank all p-values from smallest to largest
2. For each p-value, calculate: adjusted_p = p × (total_tests / rank)
3. Ensure adjusted p-values are monotonically increasing
4. Call significant if adjusted_p < 0.05

Example with 5 genes:
Gene    p-value   Rank   Adjusted p    Significant?
────    ───────   ────   ──────────    ────────────
GeneA   0.001     1      0.005         Yes
GeneB   0.008     2      0.020         Yes
GeneC   0.030     3      0.050         Yes (borderline)
GeneD   0.040     4      0.050         Yes (borderline)
GeneE   0.200     5      0.200         No

In R:

# Manual BH correction
p_values <- c(0.001, 0.008, 0.030, 0.040, 0.200)
p.adjust(p_values, method = "BH")
# [1] 0.005 0.020 0.050 0.050 0.200

# DESeq2 does this automatically in the padj column
# You never need to do it manually for DE analysis

When to Use Which

Situation                          Method
─────────                          ──────
Testing 3-10 things (ANOVA post-hoc)  Bonferroni or Tukey
Testing 100s-1000s (DE, GWAS)         Benjamini-Hochberg (FDR)
Testing millions (GWAS)               Bonferroni at 5×10⁻⁸
Exploratory analysis                  FDR < 0.10 is acceptable
Confirmatory / clinical               FDR < 0.01 or Bonferroni

The Hidden Multiple Testing Problem

Multiple testing isn't always obvious. These all count as multiple tests:

  • Testing different gene sets until one is significant
  • Trying different normalization methods and picking the one that gives the best result
  • Splitting patients into subgroups until you find a significant difference
  • Running the analysis with and without outliers and reporting whichever looks better

This is called p-hacking or data dredging. It inflates your false positive rate without any formal correction because the extra tests aren't reported. Pre-register your analysis plan to avoid this.

The one rule

In genomics, never use raw p-values for calling significant results. Always use adjusted p-values (padj, q-value, FDR). If a paper reports raw p-values on a genome-wide analysis, be very skeptical.
