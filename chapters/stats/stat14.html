Chapter 14: Experimental Design and Sample Size

~45 min Intermediate Power, Sample Size, Replication, Randomization, Blocking

Design Before You Sequence

The most important statistical decisions happen before you collect any data. A poorly designed experiment can't be rescued by clever analysis. A well-designed experiment with modest sample sizes can answer your question cleanly.

This chapter is about making those decisions correctly.

Statistical Power

Power is the probability of detecting a real effect when it exists. It's the complement of the Type II error rate.

Power = 1 - β

β = probability of missing a real effect (Type II error)
α = probability of a false positive (Type I error, usually 0.05)

Typical target: power = 0.80 (80% chance of detecting the effect)

What affects power?

Factor              Effect on power
──────              ───────────────
Sample size ↑       Power ↑ (more data = easier to detect effects)
Effect size ↑       Power ↑ (bigger effects are easier to detect)
Variability ↓       Power ↑ (less noise = clearer signal)
α threshold ↑       Power ↑ (but more false positives)
Two-tailed test     Power ↓ vs one-tailed (but more honest)

Sample Size Calculation

Before starting an experiment, calculate how many samples you need to detect the effect you care about with adequate power.

# In R:
power.t.test(
  delta = 2,        # Expected difference between means
  sd = 3,           # Expected standard deviation
  sig.level = 0.05, # Alpha
  power = 0.80      # Desired power
)
# n = 19 per group

# For RNA-Seq, use RNASeqPower or PROPER:
library(RNASeqPower)
rnapower(depth = 20, n = 3, cv = 0.4, effect = 2, alpha = 0.05)

What you need to know in advance:

  • Expected effect size: How big a fold change do you expect? Use pilot data or published literature.
  • Variability: How variable is expression between replicates? Coefficient of variation (CV) for RNA-Seq is typically 0.3-0.5.
  • Desired power: Usually 0.80. For clinical studies, 0.90.
  • Significance threshold: 0.05 for most studies. Lower for GWAS.

Replication: Technical vs Biological

Not all replicates are equal.

Technical replicates:  Same sample, processed multiple times
                       Measures technical variability (pipetting, sequencing)
                       Does NOT increase biological inference

Biological replicates: Different biological samples (different mice, patients, cultures)
                       Measures biological variability
                       REQUIRED for statistical inference
Technical replicates don't count

Sequencing the same RNA sample 3 times gives you 3 technical replicates. You cannot use these to test for differential expression. You need 3 different biological samples (different mice, different patients, different cell culture flasks). This is the most common design mistake in genomics.

Randomization

Randomly assign samples to treatment groups. This distributes unknown confounders evenly across groups.

# In R:
set.seed(42)
samples <- paste0("sample", 1:20)
treatment <- sample(rep(c("treated", "control"), each = 10))
data.frame(sample = samples, treatment = treatment)

Without randomization, you might accidentally put all the older mice in the treated group, confounding age with treatment.

Blocking

If you know about a source of variability (batch, sex, cage), use it to your advantage by blocking. Process equal numbers of treated and control samples in each batch.

Bad design (confounded):
  Batch 1: all treated samples
  Batch 2: all control samples

Good design (blocked):
  Batch 1: 3 treated + 3 control
  Batch 2: 3 treated + 3 control

Blocking removes the batch effect from your comparison. You can then include batch in the model to account for any remaining variation.

Paired Designs

When possible, use paired designs. Measure the same subject before and after treatment. This removes between-subject variability, dramatically increasing power.

Unpaired design: 10 treated mice + 10 control mice
  Between-mouse variability adds noise to the comparison

Paired design: 10 mice, each measured before and after treatment
  Each mouse serves as its own control
  Between-mouse variability is eliminated
  Much more powerful with the same n

Common Design Mistakes in Genomics

  • Too few replicates. n=2 per group is almost never enough. Aim for n≥3, ideally n≥5.
  • Confounded batch effects. All treated samples in one batch, all controls in another.
  • Technical replicates mistaken for biological. Three sequencing runs of the same sample.
  • Pseudoreplication. Treating multiple cells from the same mouse as independent samples.
  • No power calculation. Starting an experiment without knowing if you can detect the effect.
  • Unbalanced groups. 10 treated, 3 control. Reduces power and makes batch correction harder.

☐ What is the biological question?
☐ What is the primary outcome variable?
☐ What effect size is biologically meaningful?
☐ How variable is the measurement? (pilot data or literature)
☐ How many biological replicates do I need? (power calculation)
☐ Are there known confounders? (batch, sex, age)
☐ How will I randomize sample assignment?
