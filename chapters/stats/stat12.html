Chapter 12: Classification — From Logistic Regression to Random Forests

~55 min Intermediate Classification, ROC, Random Forest, Cross-validation

What is Classification?

Regression predicts a continuous number. Classification predicts a category. Is this tumor malignant or benign? Will this patient respond to treatment? Does this variant cause disease?

Classification is one of the most important tasks in clinical genomics. A good classifier can turn a gene expression profile into a diagnosis.

Logistic Regression as a Classifier

You already know logistic regression from Chapter 11. It predicts the probability of belonging to a class. To classify, you pick a threshold (usually 0.5): if predicted probability > 0.5, predict class 1.

# In R:
model <- glm(cancer ~ gene1 + gene2 + gene3,
             family = binomial, data = train_data)

# Predict probabilities on new data
probs <- predict(model, newdata = test_data, type = "response")

# Classify at 0.5 threshold
predictions <- ifelse(probs > 0.5, "cancer", "normal")

Evaluating a Classifier

Accuracy alone is misleading. If 95% of samples are healthy, a classifier that always says "healthy" has 95% accuracy but is useless.

Confusion matrix

                  Predicted
                  Positive  Negative
Actual Positive:    TP        FN
Actual Negative:    FP        TN

TP = True Positive  (correctly predicted disease)
TN = True Negative  (correctly predicted healthy)
FP = False Positive (predicted disease, actually healthy)
FN = False Negative (predicted healthy, actually disease)

Key metrics

Sensitivity (Recall) = TP / (TP + FN)   # How many true cases did you catch?
Specificity          = TN / (TN + FP)   # How many true negatives did you correctly identify?
Precision            = TP / (TP + FP)   # Of your positive predictions, how many were right?
Accuracy             = (TP + TN) / total

F1 score = 2 × (Precision × Recall) / (Precision + Recall)  # Balanced metric

ROC Curves and AUC

Instead of picking one threshold, plot sensitivity vs (1-specificity) across all possible thresholds. This is the ROC (Receiver Operating Characteristic) curve.

AUC = Area Under the ROC Curve

AUC = 0.5   Random classifier (useless)
AUC = 0.7   Acceptable
AUC = 0.8   Good
AUC = 0.9   Excellent
AUC = 1.0   Perfect (probably overfitting)
# In R:
library(pROC)
roc_obj <- roc(test_data$cancer, probs)
auc(roc_obj)
plot(roc_obj)

Decision Trees

A tree of if-then rules. Easy to interpret. "If BRCA1 expression > 500 AND TP53 < 100, predict cancer."

# In R:
library(rpart)
tree <- rpart(cancer ~ ., data = train_data, method = "class")
rpart.plot::rpart.plot(tree)

Single trees are interpretable but unstable. Small changes in data can produce very different trees. Random forests fix this.

Random Forests

Build hundreds of decision trees, each on a random subset of samples and features. Average their predictions. The result is much more stable and accurate than any single tree.

# In R:
library(randomForest)
rf <- randomForest(cancer ~ ., data = train_data, ntree = 500)

# Feature importance: which genes matter most?
importance(rf)
varImpPlot(rf)

Why random forests work well in genomics:

  • Handle thousands of features (genes) without feature selection
  • Capture non-linear relationships and interactions
  • Built-in feature importance scores
  • Robust to outliers and missing data
  • Work well even when features outnumber samples (p >> n)

Cross-Validation: Honest Performance Estimates

Never evaluate a model on the same data you trained it on. You'll get an optimistic estimate because the model has memorized the training data.

k-fold cross-validation:
1. Split data into k folds (usually k=5 or k=10)
2. Train on k-1 folds, test on the remaining fold
3. Repeat k times, each fold serves as test set once
4. Average the performance across all k folds

This gives an honest estimate of how the model will perform on new data.
# In R with caret:
library(caret)
ctrl <- trainControl(method = "cv", number = 10)
model <- train(cancer ~ ., data = df,
               method = "rf",
               trControl = ctrl)
model$results

Overfitting: The Biggest Danger

A model that's too complex memorizes the training data, including its noise. It performs perfectly on training data but poorly on new data.

Signs of overfitting:
- Training accuracy >> test accuracy
- AUC on training set >> AUC on test set
- Model has more parameters than samples
- Adding more features keeps improving training performance

Solutions:
- Cross-validation to detect it
- Regularization (Ridge, Lasso) to penalize complexity
- Simpler models
- More data

Classification in Bioinformatics

  • Cancer subtype classification from gene expression (PAM50 for breast cancer)
  • Variant pathogenicity prediction (CADD, REVEL scores use random forests)
  • Cell type annotation in single-cell RNA-Seq
  • Treatment response prediction from tumor genomics
  • Splice site prediction from sequence features
