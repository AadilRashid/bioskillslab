Chapter 8: Comparing Two Groups — t-tests and Beyond

~40 min Intermediate t-test, Paired, Unpaired, Wilcoxon, Assumptions

The Simplest Question in Statistics

Are these two groups different? Treated vs control. Tumor vs normal. Wildtype vs knockout. You have measurements from both groups and you want to know if the difference is real or just noise.

The t-test is the workhorse for this. It's been around since 1908 and it's still the right tool for most two-group comparisons.

Unpaired (Independent) t-test

Use this when the two groups are independent. Different mice, different patients, different cell cultures.

# In R:
t.test(expression ~ group, data = df)

# Or explicitly:
t.test(treated_values, control_values)

# Example output:
#   t = 3.42, df = 8, p-value = 0.009
#   95% CI: [1.2, 6.8]
#   mean of treated: 12.5, mean of control: 8.5

What the t-test actually computes

t = (mean₁ - mean₂) / SE_difference

SE_difference = √(s₁²/n₁ + s₂²/n₂)

The t-statistic is just the difference between means divided by how much you'd expect that difference to vary by chance. Big t = big difference relative to noise = small p-value.

Paired t-test

Use this when measurements are naturally paired. Same patient before and after treatment. Same cell line with and without drug. Same tissue, two different stains.

Pairing removes between-subject variability. If Patient A has naturally high expression and Patient B has naturally low expression, pairing accounts for that. You're only looking at the within-subject change.

# In R:
t.test(before, after, paired = TRUE)

# This is equivalent to a one-sample t-test on the differences:
differences <- after - before
t.test(differences, mu = 0)
When to use paired vs unpaired

If you can draw a line connecting each observation in group 1 to a specific observation in group 2, use paired. If the observations are independent, use unpaired. Using unpaired when data is paired throws away information and reduces your power.

Assumptions of the t-test

  1. Data is continuous (not counts or categories)
  2. Observations are independent (for unpaired) or properly paired
  3. Data is approximately Gaussian in each group
  4. Variances are roughly equal between groups (for classic t-test; Welch's t-test relaxes this)

What if assumptions are violated?

Problem                          Solution
──────                           ────────
Data is skewed                   Log-transform, then t-test
Data is not Gaussian             Wilcoxon rank-sum test (nonparametric)
Unequal variances                Welch's t-test (default in R)
Data is counts                   Use a count-based test (DESeq2, not t-test)
Very small sample (n < 5)        Consider nonparametric or exact tests

Welch's t-test

The standard t-test assumes equal variances. Welch's version doesn't. It adjusts the degrees of freedom to account for unequal variances. In R, t.test() uses Welch's by default. There's almost never a reason to use the classic version.

Wilcoxon Rank-Sum Test (Mann-Whitney U)

The nonparametric alternative to the unpaired t-test. Instead of comparing means, it compares the ranks of the values. Doesn't assume Gaussian data.

# In R:
wilcox.test(treated_values, control_values)

# Paired version (Wilcoxon signed-rank):
wilcox.test(before, after, paired = TRUE)

Use this when your data is clearly non-Gaussian and log-transformation doesn't help, or when you have ordinal data (rankings, scores).

t-tests in Bioinformatics

When you DO use t-tests

  • Comparing a single gene's expression between two groups (qPCR data)
  • Comparing a summary metric (total read count, mapping rate) between batches
  • Comparing protein abundance between conditions (mass spec)

When you DON'T use t-tests

  • RNA-Seq differential expression. Use DESeq2 or edgeR. They use the negative binomial model, not t-tests. Running t-tests on RNA-Seq counts gives wrong results.
  • Comparing proportions. Use Fisher's exact test or chi-squared.
  • More than two groups. Use ANOVA (Chapter 9).
Don't run 20,000 t-tests on RNA-Seq data

Some people run a t-test on each gene independently. This ignores the count nature of the data, doesn't share information across genes (which DESeq2 does to stabilize variance estimates), and gives unreliable results for low-count genes. Use the right tool.

Choosing the Right Test: Decision Tree

Comparing two groups?
├── Data is paired?
│   ├── Yes → Gaussian? → Paired t-test
│   │                  → Not Gaussian? → Wilcoxon signed-rank
│   └── No (independent)
│       ├── Gaussian? → Welch's t-test
│       └── Not Gaussian? → Wilcoxon rank-sum (Mann-Whitney)
│
Comparing more than two groups? → ANOVA (Chapter 9)
