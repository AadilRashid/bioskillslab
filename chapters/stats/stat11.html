Why One Predictor is Never Enough

Gene expression doesn't depend on just one thing. It depends on treatment, age, sex, batch, cell type, and dozens of other factors simultaneously. Simple regression handles one predictor. Multiple regression handles all of them at once.

More importantly, multiple regression lets you ask: what is the effect of treatment, after accounting for age and sex? This is the question that matters in real biology.

Multiple Linear Regression

y = β₀ + β₁x₁ + β₂x₂ + β₃x₃ + ... + ε

Example:
expression = β₀ + β₁(treatment) + β₂(age) + β₃(sex) + ε

In R:

model <- lm(expression ~ treatment + age + sex, data = df)
summary(model)

# Coefficients:
#                  Estimate Std. Error t value Pr(>|t|)
# (Intercept)       85.2      12.1     7.04   0.0001 ***
# treatmentDrug      18.5       3.2     5.78   0.0002 ***
# age                 0.8       0.4     2.00   0.065  .
# sexMale            -5.2       2.8    -1.86   0.082  .
#
# R-squared: 0.72

Each coefficient tells you the effect of that variable holding all others constant. The drug increases expression by 18.5 units, after accounting for age and sex differences.

Confounders: Why This Matters

A confounder is a variable that's associated with both your predictor and your outcome. If you don't include it in the model, you get a biased estimate.

Example: You find that older patients have higher tumor expression.
But older patients also received more prior treatments.
Prior treatment is a confounder.

Without adjustment: age effect = 3.2 (biased, includes treatment effect)
With adjustment:    age effect = 0.8 (true age effect)

In bioinformatics:

  • Batch effects confound treatment comparisons. Include batch in the model.
  • Cell type composition confounds bulk RNA-Seq. Deconvolve or include as covariate.
  • Sequencing depth confounds expression comparisons. Normalize first.
  • Age and sex affect gene expression. Include in DESeq2 design formula.

The DESeq2 Design Formula is Multiple Regression

# Simple: just test condition
dds <- DESeqDataSetFromMatrix(counts, coldata, design = ~condition)

# With covariates: adjust for batch and sex before testing condition
dds <- DESeqDataSetFromMatrix(counts, coldata, design = ~batch + sex + condition)

# The last term is what you're testing.
# Earlier terms are adjusted for (like covariates in multiple regression).

Multicollinearity: When Predictors Correlate

If two predictors are highly correlated with each other, the model can't separate their effects. This is multicollinearity.

Example: You include both "age" and "years_since_diagnosis" in a model.
These are highly correlated. The model can't tell which one is driving the effect.
Both coefficients become unstable and unreliable.

Solution: Remove one of the correlated predictors, or use PCA to create
uncorrelated composite variables.

Generalized Linear Models (GLMs)

Linear regression assumes the outcome is continuous and Gaussian. GLMs extend this to other types of outcomes.

Outcome type          Distribution    Link function    Use case
────────────          ────────────    ─────────────    ────────
Continuous            Gaussian        Identity         Gene expression (log-transformed)
Binary (0/1)          Binomial        Logit            Disease yes/no, variant present/absent
Counts                Poisson         Log              Read counts (simplified)
Counts (overdispersed) Neg. Binomial  Log              RNA-Seq (DESeq2, edgeR)
Survival time         Weibull/Cox     Log              Patient survival

Logistic Regression

The most important GLM for bioinformatics. Predicts a binary outcome (disease/healthy, variant/no variant, responder/non-responder).

# In R:
model <- glm(disease ~ gene_expression + age + sex,
             family = binomial(link = "logit"),
             data = df)
summary(model)

# Coefficients are on the log-odds scale.
# Exponentiate to get odds ratios:
exp(coef(model))

# Interpretation:
# OR = 2.3 for gene_expression means:
# Each unit increase in expression multiplies the odds of disease by 2.3

Odds ratios in GWAS

GWAS reports odds ratios for each SNP. OR = 1.5 means carrying the risk allele multiplies your odds of disease by 1.5. OR = 0.7 means it's protective (reduces odds by 30%).

Model Selection: How Many Predictors?

More predictors always improve R² on your training data. But too many predictors leads to overfitting: the model fits noise instead of signal and performs poorly on new data.

Rule of thumb: you need at least 10-20 observations per predictor.
With 50 patients, don't include more than 2-5 predictors.

Criteria for comparing models:
AIC (Akaike Information Criterion): lower is better, penalizes complexity
BIC (Bayesian Information Criterion): stricter penalty for complexity
Adjusted R²: R² penalized for number of predictors
# In R:
model1 <- lm(y ~ x1, data = df)
model2 <- lm(y ~ x1 + x2, data = df)
AIC(model1, model2)   # Compare models
