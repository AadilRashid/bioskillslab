Chapter 8: Biological Databases

~50 min Beginner NCBI, Ensembl, UniProt, UCSC, SRA, Data Integrity

The Big Four Databases

Bioinformatics revolves around public databases. Knowing which database to use — and how to query it — is a core skill.

1. NCBI — National Center for Biotechnology Information

The US hub for biological data. NCBI hosts dozens of interconnected databases:

  • GenBank — Nucleotide sequences (genes, genomes, mRNA). Over 2 billion sequences
  • SRA (Sequence Read Archive) — Raw sequencing reads. Over 70 petabytes. This is where you download FASTQ files from published studies
  • GEO (Gene Expression Omnibus) — Processed expression datasets (microarray, RNA-Seq)
  • PubMed — 35+ million biomedical literature citations
  • dbSNP — Known genetic variants (SNPs, indels)
  • ClinVar — Clinically relevant variants and their disease associations
  • RefSeq — Curated reference sequences (higher quality than GenBank)
  • Gene — Gene-centric information aggregated from multiple sources

2. Ensembl (EMBL-EBI)

The European genome annotation resource. Best for:

  • Downloading reference genomes (FASTA) and gene annotations (GTF/GFF)
  • Comparative genomics across species
  • Variant Effect Predictor (VEP) for annotating variants
  • BioMart for bulk data queries

3. UniProt

The definitive protein database. Two sections:

  • Swiss-Prot — Manually curated, high quality (~570,000 entries)
  • TrEMBL — Automatically annotated, much larger (~250 million entries)

Use UniProt for protein function, domains, structure, interactions, and disease associations.

4. UCSC Genome Browser

A visual genome exploration tool with hundreds of annotation tracks. Excellent for:

  • Visually inspecting a genomic region
  • Viewing conservation across species
  • Downloading annotation tables via Table Browser
  • Custom tracks (upload your own BED/BAM files)

Downloading Data

wget and curl

# Download a file with wget
wget https://ftp.ensembl.org/pub/release-110/fasta/homo_sapiens/dna/Homo_sapiens.GRCh38.dna.primary_assembly.fa.gz

# Resume interrupted download
wget -c https://example.com/large_file.gz

# Download with curl
curl -O https://example.com/file.gz

# Download multiple files listed in a text file
wget -i urls.txt

Downloading from SRA

# Install SRA Toolkit
conda install -c bioconda sra-tools

# Method 1: prefetch + fastq-dump (traditional)
prefetch SRR1234567
fastq-dump --split-files --gzip SRR1234567

# Method 2: fasterq-dump (much faster, recommended)
fasterq-dump SRR1234567 --threads 4 --split-files
# Then compress (fasterq-dump doesn't output gzip)
gzip SRR1234567_1.fastq SRR1234567_2.fastq

# Download multiple accessions
while read acc; do
    fasterq-dump "$acc" --threads 4 --split-files
done < accession_list.txt

rsync and scp

# rsync — efficient for large transfers (only copies differences)
rsync -avz --progress user@server:/data/results/ local_results/

# scp — simple secure copy
scp user@server:/data/sample.bam ./
scp -r user@server:/data/project/ ./local_project/

Data Integrity — Verify Everything

Downloaded files can be corrupted by network errors, disk issues, or incomplete transfers. Always verify with checksums.

# Generate checksums
md5sum file.fa.gz                    # MD5 hash
sha256sum file.fa.gz                 # SHA-256 hash (more secure)

# Verify against a published checksum
echo "abc123def456  file.fa.gz" | md5sum -c
# Output: file.fa.gz: OK

# Ensembl provides CHECKSUMS files — always use them
wget https://ftp.ensembl.org/pub/release-110/fasta/homo_sapiens/dna/CHECKSUMS
md5sum -c CHECKSUMS
Corrupted Data = Corrupted Results

A truncated FASTQ file will silently produce fewer alignments. A corrupted reference genome will cause wrong variant calls. Always verify checksums after downloading. It takes seconds and can save weeks of wasted analysis.

Programmatic Access with Biopython Entrez

from Bio import Entrez, SeqIO

Entrez.email = "your@email.com"  # Required by NCBI

# Search for BRCA1 sequences in human
handle = Entrez.esearch(db="nucleotide", 
                        term="BRCA1[Gene] AND Homo sapiens[Organism] AND RefSeq[Filter]",
                        retmax=5)
record = Entrez.read(handle)
print(f"Found {record['Count']} results")

# Fetch a specific sequence by accession
handle = Entrez.efetch(db="nucleotide", id="NM_007294", rettype="fasta", retmode="text")
seq_record = SeqIO.read(handle, "fasta")
print(f"{seq_record.id}: {len(seq_record.seq)} bp")

# Search PubMed
handle = Entrez.esearch(db="pubmed", term="CRISPR AND 2024[pdat]", retmax=10)
results = Entrez.read(handle)
print(f"Found {results['Count']} CRISPR papers from 2024")

Compressed Data

# gzip — standard compression for bioinformatics
gzip file.fastq                  # Compress → file.fastq.gz (original deleted)
gzip -k file.fastq               # Keep original
gunzip file.fastq.gz             # Decompress

# View compressed files without decompressing
zcat file.fastq.gz | head -20    # Linux
gzcat file.fastq.gz | head -20   # macOS

# Many tools read gzipped files directly
bwa mem ref.fa reads_R1.fastq.gz reads_R2.fastq.gz > aligned.sam
grep -c "^>" <(zcat sequences.fasta.gz)
Storage Tip

FASTQ files compress ~3-4x with gzip. A 50 GB FASTQ becomes ~15 GB. Always store compressed. Most bioinformatics tools accept .gz input directly.
