Chapter 4: Advanced Shell & Scripting

~75 min Intermediate Bash, Loops, find, xargs, Makefiles

Why Shell Scripts?

You've learned individual Unix commands. Now it's time to combine them into reusable, automated scripts. In bioinformatics, you'll process dozens or hundreds of samples through the same pipeline. Doing this manually is slow, error-prone, and not reproducible. Shell scripts solve all three problems.

Writing Robust Bash Scripts

Every Bash script should start with these two lines:

#!/bin/bash
set -euo pipefail

What does set -euo pipefail do?

  • -e — Exit immediately if any command fails (non-zero exit code)
  • -u — Treat undefined variables as errors (catches typos like $SAMPL instead of $SAMPLE)
  • -o pipefail — If any command in a pipe fails, the whole pipe fails (without this, only the last command's exit code matters)
Without set -euo pipefail

Your script will silently continue after errors. Imagine BWA fails but samtools still runs on an empty file — you'd get empty results with no error message. This is how bad papers happen.

Variables and Arguments

#!/bin/bash
set -euo pipefail

# Variables (no spaces around =)
SAMPLE="sample1"
THREADS=8
REF="data/reference/GRCh38.fa"
OUTDIR="results/alignments"

# Create output directory if it doesn't exist
mkdir -p "${OUTDIR}"

# Use variables with ${} (braces prevent ambiguity)
echo "Processing ${SAMPLE} with ${THREADS} threads..."
bwa mem -t ${THREADS} ${REF} data/raw/${SAMPLE}_R1.fastq data/raw/${SAMPLE}_R2.fastq \
    | samtools sort -@ ${THREADS} -o ${OUTDIR}/${SAMPLE}.bam
samtools index ${OUTDIR}/${SAMPLE}.bam

Command-Line Arguments

#!/bin/bash
set -euo pipefail

# $1, $2, etc. are positional arguments
# $# = number of arguments, $@ = all arguments
SAMPLE=$1
THREADS=${2:-4}    # Default to 4 if not provided

if [ $# -lt 1 ]; then
    echo "Usage: $0 <sample_name> [threads]" >&2
    exit 1
fi

echo "Aligning ${SAMPLE} with ${THREADS} threads"
# ... rest of pipeline
# Run it:
bash align.sh sample1 8
bash align.sh sample2        # Uses default 4 threads

Conditionals

# Check if file exists before processing
if [ -f "data/raw/${SAMPLE}.fastq" ]; then
    echo "Found ${SAMPLE}, processing..."
    fastqc "data/raw/${SAMPLE}.fastq" -o results/qc/
else
    echo "ERROR: data/raw/${SAMPLE}.fastq not found" >&2
    exit 1
fi

# Check if output already exists (skip if done)
if [ ! -f "results/${SAMPLE}.bam" ]; then
    echo "Aligning ${SAMPLE}..."
    bwa mem ref.fa data/${SAMPLE}.fastq | samtools sort -o results/${SAMPLE}.bam
else
    echo "${SAMPLE}.bam already exists, skipping."
fi

# Check if a directory exists
[ -d "results/qc" ] || mkdir -p results/qc

# Common test operators:
# -f file    File exists and is regular file
# -d dir     Directory exists
# -s file    File exists and is non-empty
# -z "$var"  Variable is empty
# -n "$var"  Variable is non-empty

Loops — Batch Processing

This is where shell scripting becomes essential. Process all your samples with a single script:

for Loops

# Process all FASTQ files in a directory
for fq in data/raw/*.fastq; do
    sample=$(basename "$fq" .fastq)    # Extract filename without path/extension
    echo "Running FastQC on ${sample}..."
    fastqc "$fq" -o results/qc/
done

# Process a list of sample names
for sample in control_1 control_2 treatment_1 treatment_2; do
    echo "Aligning ${sample}..."
    bwa mem -t 8 ref.fa data/${sample}_R1.fastq data/${sample}_R2.fastq \
        | samtools sort -o results/${sample}.bam
    samtools index results/${sample}.bam
done

# Process samples from a file (one per line)
while IFS= read -r sample; do
    echo "Processing ${sample}"
    # ... pipeline commands
done < samples.txt
basename and dirname 
basename /home/user/data/sample1.fastq          # sample1.fastq
basename /home/user/data/sample1.fastq .fastq   # sample1
dirname  /home/user/data/sample1.fastq          # /home/user/data

Command Substitution

# Capture command output in a variable
NUM_READS=$(grep -c "^@" data/sample.fastq)
echo "Found ${NUM_READS} reads"

# Date-stamped output directory
OUTDIR="results/$(date +%Y%m%d)"
mkdir -p "${OUTDIR}"

# Arithmetic
TOTAL_LINES=$(wc -l < reads.fastq)
NUM_READS=$((TOTAL_LINES / 4))
echo "${NUM_READS} reads in file"

# Use in conditionals
if [ $(grep -c "^>" seqs.fa) -eq 0 ]; then
    echo "ERROR: No sequences found" >&2
    exit 1
fi

find and xargs — Processing Many Files

find locates files. xargs runs commands on them. Together they're incredibly powerful.

find

# Find all FASTQ files recursively
find data/ -name "*.fastq" -type f

# Find files modified in the last 24 hours
find results/ -mtime -1 -type f

# Find large files (>1GB)
find data/ -size +1G -type f

# Find empty files (potential errors)
find results/ -empty -type f

# Find and delete temporary files
find . -name "*.tmp" -type f -delete

# Combine conditions
find data/ -name "*.fastq" -size +100M -type f

xargs

# Run FastQC on all FASTQ files found by find
find data/ -name "*.fastq" | xargs -I{} fastqc {} -o results/qc/

# Parallel processing (4 jobs at once)
find data/ -name "*.fastq" | xargs -P 4 -I{} fastqc {} -o results/qc/

# Count lines in all BED files
find results/ -name "*.bed" | xargs wc -l

# Safer with filenames containing spaces (use -print0 and -0)
find data/ -name "*.fastq" -print0 | xargs -0 -I{} fastqc {}

Makefiles — Dependency-Based Pipelines

Make tracks dependencies between files. It only re-runs steps whose inputs have changed. This saves hours when you modify one step of a pipeline.

# Makefile (IMPORTANT: indentation MUST be tabs, not spaces)
SAMPLES = sample1 sample2 sample3
BAMS = $(patsubst %,results/%.bam,$(SAMPLES))

.PHONY: all clean

all: $(BAMS) results/multiqc_report.html

# Rule: BAM depends on FASTQ and reference
results/%.bam: data/raw/%.fastq data/reference/genome.fa
	@echo "Aligning $*..."
	bwa mem -t 8 data/reference/genome.fa $< | samtools sort -o $@
	samtools index $@

# Rule: MultiQC depends on all BAMs
results/multiqc_report.html: $(BAMS)
	multiqc results/ -o results/

clean:
	rm -f results/*.bam results/*.bai results/multiqc_report.html

Key Make concepts:

  • $@ — The target (output file)
  • $< — The first prerequisite (first input file)
  • $^ — All prerequisites
  • $* — The stem (the % part of the pattern)
  • % — Wildcard pattern (like * in shell)
# Run the pipeline
make                    # Build everything
make -n                 # Dry run (show what would happen)
make -j 4               # Run up to 4 jobs in parallel
make clean              # Remove all outputs
make results/sample1.bam  # Build just one target
Make vs Snakemake

Make is great for simple pipelines. For complex bioinformatics workflows with many samples and steps, Snakemake (Chapter 14) is more powerful — it supports wildcards, cluster submission, conda integration, and more.

Practical Example: Complete QC Script

#!/bin/bash
set -euo pipefail

# Configuration
DATADIR="data/raw"
OUTDIR="results/qc"
THREADS=4

# Validate
if [ ! -d "${DATADIR}" ]; then
    echo "ERROR: ${DATADIR} not found" >&2
    exit 1
fi

mkdir -p "${OUTDIR}"

# Count input files
N_FILES=$(find "${DATADIR}" -name "*.fastq" | wc -l)
echo "Found ${N_FILES} FASTQ files in ${DATADIR}"

if [ "${N_FILES}" -eq 0 ]; then
    echo "ERROR: No FASTQ files found" >&2
    exit 1
fi

# Run FastQC on all files (parallel)
echo "Running FastQC with ${THREADS} parallel jobs..."
find "${DATADIR}" -name "*.fastq" | xargs -P ${THREADS} -I{} fastqc {} -o "${OUTDIR}"

# Aggregate with MultiQC
echo "Running MultiQC..."
multiqc "${OUTDIR}" -o "${OUTDIR}" --force

echo "Done! Reports in ${OUTDIR}/"
